<objective>
Build `scripts/tile-photos.js`, the bulk DZI tile pipeline that composes the Plan 01 libraries (`tile-config.json`, `downloadSharedFile`, `advanceStatus`) with `vips dzsave` to advance each eligible manifest row from `status: discovered` (or `status: downloaded`) to `status: tiled`.

Purpose: TILE-01 + TILE-02 + TILE-03 — the script is the deliverable that "produces a DZI tile pyramid from each downloaded TIFF on the datacenter server", idempotently, with parameters reproducible from the committed config.

Output: `scripts/tile-photos.js` CLI, `scripts/tile-photos.test.js` covering path computation + idempotency, npm alias `photos:tile`, plus `scripts/tile-photos.test.js` added to the `npm test` script glob.

Phase 26 gap closure: the ROADMAP goal mentions "manifest row in `status: downloaded`" but Phase 26 only marks rows `status: discovered`. This plan closes the gap by making `tile-photos.js` handle BOTH transitions: `discovered → downloaded → tiled` for rows missing TIFFs, and `downloaded → tiled` for rows whose TIFFs are already cached. Both transitions are recorded in the manifest with distinct statuses so the run is resumable at either boundary.
</objective>
