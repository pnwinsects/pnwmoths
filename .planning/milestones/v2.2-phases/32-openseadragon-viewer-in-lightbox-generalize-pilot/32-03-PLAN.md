<objective>
Add per-specimen thumbnail images to the tiling and upload pipeline, then backfill thumbnails for the two pilot specimens (abagrotis-apposita A-D and A-V) and wire the template to use them.

The core problem: DZI tile levels are powers-of-two and don't produce a usable carousel thumbnail at any single level (level 0 = 1×1 px; level 8 = 256 px max; there is no level that gives a clean ~376×249 image for all source sizes). The fix: generate a single-file thumbnail WebP during tiling (`vips thumbnail` at thumbnailWidth=1500), upload it alongside the DZI pyramid, and serve it via Bunny's image optimizer for display-side resizing.

Output: pipeline change ships (tile + upload scripts updated); pilot thumbnails accessible on CDN; species.njk carousel uses thumbnail URLs; build passes.
</objective>

## Task 2 — Add `runVipsThumbnail` to tile-photos.js

After `runVipsDzsave(cachePath, prefix, config)` in the tile stage of `main()`, call a new helper:

```js
function runVipsThumbnail(sourceTiff, prefix, config) {
  execFileSync('vips', [
    'thumbnail',
    sourceTiff + '[unlimited]',
    `${prefix}_thumbnail.webp`,
    String(config.thumbnailWidth),
  ], { stdio: ['pipe', 'pipe', 'pipe'] });
}
```

Call immediately after `runVipsDzsave`:
```js
runVipsDzsave(cachePath, prefix, config);
runVipsThumbnail(cachePath, prefix, config);
```

The `[unlimited]` flag is required for large TIFFs (matches the dzsave pattern already in use). `vips thumbnail` produces a single WebP at the specified width, height auto-computed to preserve aspect ratio.

**Acceptance:** `grep -n 'runVipsThumbnail' scripts/tile-photos.js` shows the function definition and call site.

## Task 3 — Add thumbnail upload + cleanup to upload-tiles.js

In the per-row upload loop in `main()`, after uploading `_files/` tiles and before uploading the `.dzi` descriptor, add:

```js
// --- Upload thumbnail ---
const thumbnailLocalPath = `${localPrefix}_thumbnail.webp`;
const thumbnailUrl = `${storageBase}_thumbnail.webp`;
if (existsSync(thumbnailLocalPath)) {
  const thumbArgs = [
    '-s', '-S', '-f',
    '-X', 'PUT',
    '-H', `AccessKey: ${BUNNY_API_KEY}`,
    '-H', 'Content-Type: image/webp',
    '--data-binary', `@${thumbnailLocalPath}`,
    thumbnailUrl,
  ];
  await withRetry(
    () => execFileSync('curl', thumbArgs, { stdio: ['pipe', 'pipe', 'inherit'] }),
    `upload ${pair}/_thumbnail`
  );
}
```

In the post-upload cleanup (after `advanceStatus(row, 'uploaded')`), add:
```js
if (existsSync(thumbnailLocalPath)) await unlink(thumbnailLocalPath);
```

The `existsSync` guard means rows tiled before this change (no thumbnail on disk) are silently skipped.

**Acceptance:** `grep -n '_thumbnail' scripts/upload-tiles.js` shows upload and cleanup lines.

## Task 4 — Backfill pilot thumbnails

The two pilot specimens (abagrotis-apposita A-D and A-V) were tiled and uploaded before thumbnail generation existed. Re-download the TIFFs, generate thumbnails, upload directly to Bunny.

Save as a temporary script `scripts/backfill-thumbnails.mjs` and run it:

```js
import { execFileSync } from 'node:child_process';
import { existsSync, readFileSync } from 'node:fs';
import { mkdir, unlink } from 'node:fs/promises';
import { downloadSharedFile } from './lib/dropbox-download.js';

const config = JSON.parse(readFileSync('scripts/tile-config.json', 'utf8'));
const DROPBOX_TOKEN = process.env.DROPBOX_TOKEN;
const BUNNY_API_KEY = process.env.BUNNY_API_KEY;
const BUNNY_STORAGE_HOST = process.env.BUNNY_STORAGE_HOST ?? 'la.storage.bunnycdn.com';
const BUNNY_ZONE = 'pnwmoths';

const specimens = [
  { slug: 'abagrotis-apposita', id: 'A', view: 'D', dropbox_path: '/Abagrotis apposita-A-D.tif', content_hash: 'e6f2267971160000' },
  { slug: 'abagrotis-apposita', id: 'A', view: 'V', dropbox_path: '/Abagrotis apposita-A-V.tif', content_hash: 'bc92a518d4be0000' },
];

await mkdir('var/tiffs', { recursive: true });

for (const s of specimens) {
  const pair = `${s.id}-${s.view}`;
  const tiffPath = `var/tiffs/${s.content_hash}-${pair}.tif`;
  const thumbPath = `var/tiffs/${pair}_thumbnail.webp`;

  if (!existsSync(tiffPath)) {
    console.log(`Downloading ${pair}...`);
    await downloadSharedFile({
      shareUrl: config.dropboxShareUrl,
      dropboxPath: s.dropbox_path,
      token: DROPBOX_TOKEN,
      destPath: tiffPath,
    });
  }

  console.log(`Generating thumbnail for ${pair}...`);
  execFileSync('vips', ['thumbnail', tiffPath + '[unlimited]', thumbPath, String(config.thumbnailWidth)],
    { stdio: 'inherit' });

  const url = `https://${BUNNY_STORAGE_HOST}/${BUNNY_ZONE}/species-tiles/${s.slug}/${pair}_thumbnail.webp`;
  console.log(`Uploading to ${url}...`);
  execFileSync('curl', ['-s', '-S', '-f', '-X', 'PUT',
    '-H', `AccessKey: ${BUNNY_API_KEY}`,
    '-H', 'Content-Type: image/webp',
    '--data-binary', `@${thumbPath}`, url], { stdio: 'inherit' });

  await unlink(tiffPath);
  await unlink(thumbPath);
  console.log(`Done: ${pair}`);
}
```

Run:
```bash
DROPBOX_TOKEN=... BUNNY_API_KEY=... node scripts/backfill-thumbnails.mjs
```

Verify both thumbnails are accessible:
```bash
curl -I "https://pnwmoths.b-cdn.net/species-tiles/abagrotis-apposita/A-D_thumbnail.webp"
curl -I "https://pnwmoths.b-cdn.net/species-tiles/abagrotis-apposita/A-V_thumbnail.webp"
```

Delete the backfill script after use (`git rm scripts/backfill-thumbnails.mjs` or just don't commit it).

**Acceptance:** Both curl -I commands return HTTP 200.

- Bunny Optimizer: once thumbnails are on CDN, `?width=376` can be appended for display-side resizing if the optimizer is enabled. The template uses the raw URL for now; this is forward-compatible.
- The `existsSync` guard in Task 3 means future uploads of already-tiled rows (without thumbnail files on disk) won't fail — safe to deploy before the backfill if needed.
