<objective>
Fill the currently-empty high-res slot in `src/species/species.njk` with one `<figure>` per specimen so that the Phase 28 OSD wiring generalizes to every `high_res_available: true` species. Each figure carries the level-0 DZI tile as the static `<img>` (no-JS fallback + Lit-component carousel/thumbnail source), an `onerror` gray-placeholder fallback (D-05), and a `Specimen {id} · Dorsal|Ventral` caption.

Purpose: Phase 31 left the high-res template branch empty. Without this branch, high-res species render zero figures, the Lit component sees `_images.length === 0`, falls through to `<slot>`, and the OSD lightbox never opens. This plan unblocks VIEWER-01 generalization across all high-res species without touching the carousel/component code.

Output: A single `{% elif highRes %}` branch in `species.njk` that materializes high-res specimen figures in the same array order as `highResEntry.specimens`, each with an `onerror` placeholder per D-05.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01-01 | Tampering | `data/species-photos.json` interpolated into `<img src>` | accept | File is a committed build artifact produced by `photos:materialize` from the operator-controlled manifest. No user-input path; tampering = repo compromise, out of scope. |
| T-32-01-02 | Tampering / XSS | `specimen.specimen_id` and `specimen.view` rendered in `<figcaption>` text | mitigate | Nunjucks `{{ }}` interpolation auto-escapes HTML. `specimen_id` values come from the filename parser (Phase 26) which restricts to `[A-Za-z0-9_]+` (single letter or `OSAC_*`/`WWUC*` accession); `view` is `D` or `V` from the parser's restricted enum. No raw HTML or attribute-context injection vector. |
| T-32-01-03 | Information Disclosure | level-0 tile URL exposes tile pyramid path structure | accept | Tile pyramids are public CDN assets already served from `pnwmoths.b-cdn.net`; URL structure is the same convention used by Phase 28 pilot. No secret to leak. |
| T-32-01-04 | Denial of Service | empty or malformed `specimens` array | accept | `{% for %}` over an empty array renders nothing — falls through to no figures rendered, component falls back to `<slot>`. No crash, no infinite loop. Build-time data is validated by Phase 31's `generate-species-photos.js`. |
| T-32-01-05 | Tampering / XSS | inline `onerror` attribute on `<img>` (D-05) | accept | Attribute body is a literal string (`this.style.background='#ddd'`) authored in the template — no interpolation, no user-supplied input reaches the JS payload.
