# Milestone v1.1: Visual Identity

**Status:** ✅ SHIPPED 2026-04-18
**Phases:** 6

## Overview

Applied the visual identity of pnwmoths.biol.wwu.edu to all ~700 generated pages. Also addressed post-v1.0 contributor tooling and data quality improvements through quick tasks. The milestone established Pico CSS design token overrides as the theming pattern, a post-Vite asset copy pipeline for static files, and slug-as-foreign-key as the canonical data linking convention.

## Milestone Summary

**Key Decisions:**

- Extended `scripts/copy-images.js` for post-Vite asset copy (passthrough rules don't survive Vite's output directory rename)
- Pico CSS design token overrides via `theme.css` rather than modifying Pico source — clean separation
- `base.njk` as the single layout entry point for visual identity — one change, all pages updated
- `species_slug` as foreign key instead of `species_id` — slug is stable and human-readable; id is an implementation detail

**Issues Resolved:**

- Passthrough copy assets (CSS, images) not surviving Vite build step — fixed by extending post-Vite copy script
- `| url` filter double-prefix issue on Vite-processed assets — fixed by removing filter from Vite-bundled asset paths
- Similar species showing slugs instead of display names — fixed with name lookup + missing species guard
- Stale contributor doc paths (`content/species/` → `src/content/species/`) — fixed in quick task 260412-qrt

**Issues Deferred:**

- WR-02: Pagefind `<link>` stylesheet in body instead of `<head>` (FOUC) — deferred to v1.2
- WR-03: DuckDB instance not closed in glossary.js (resource leak) — deferred to v1.2
- WR-04: Missing ENOENT guard in check-page-weight.js — deferred to v1.2
- WR-05: Pagefind CSS path double-prefix — out of scope for v1.1
- MAINT-03: Build time <5 min on GitHub Actions — requires live CI observation

**Technical Debt Incurred:**

- Phase 6 visual verification (Task 3) was a human checkpoint; no automated visual regression test exists
- No automated test for banner image path or CSS loading across all pages
