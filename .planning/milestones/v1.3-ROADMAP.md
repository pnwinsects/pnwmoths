## Overview

Replaced the static browse pages with an interactive accordion browse page (Family → Subfamily → Genus → Species) with navigation images and client-side state filtering. ## Milestone Summary

**Key Decisions:**
- Use JSON (not Parquet) for species-×-state distribution — at 700 species × ~6 states (~4,200 pairs, ~20–30 KB), hyparquet overhead not justified
- Lit accordion must use light DOM (`createRenderRoot() { return this; }`) — Pico CSS element selectors don't penetrate shadow DOM
- DuckDB `nullstr = ''` required on both read_csv calls — blank subfamily must arrive as null, not empty string
- Taxonomy JSON embedded as `<script type="application/json" id="taxon-data">` sibling (not `data-taxonomy` attribute) — avoids HTML entity encoding issues
- Vite HTML transformer double-prefix issue: use raw `/images/...` paths and let Vite add base (not `| url` filter in templates)

**Issues Resolved:**
- Phase 11 UAT: No subfamily data in current species.csv — component correctly flattens genera directly under family
- Phase 11 UAT: Vite `| url` double-prefix on asset URLs — fixed by using raw paths
- Phase 11 UAT: Toolbar baseline misalignment — fixed via `align-items: baseline` in CSS
- Phase 11 UAT: JSON data attribute HTML entity encoding — fixed via `| safe` on `tojson` output
- Phase 11 UAT: species-states.json not emitting during `eleventy --serve` — fixed by adding writeBundle and serve-mode hooks

**Issues Deferred:**
- BROWSE-08: Images collapse when drilling into child taxon — deferred to future milestone
- BROWSE-09: Show/hide toggle state persists across reloads (sessionStorage) — deferred
- FILT-03: County filter — deferred

**Technical Debt Incurred:**

- No automated visual regression tests for visual identity (carried from v1.2)
