## Overview

Replace existing low-res species photos with OpenSeadragon deep-zoom high-res photos sourced from a ~200 GB Dropbox folder, via a resumable server-side processing pipeline. Includes Dropbox ingest + filename parser, synonym curation tooling, an end-to-end vertical-slice pilot on one species before bulk commit, libvips DZI tile generation, bunny.net bulk upload, Eleventy build integration, and generalized OSD viewer wiring across all `high_res_available: true` species.

## Milestone Summary

**Key Decisions:**
- Local manifest (species-photos-manifest.csv) as source of truth — durable per-image status, survives restarts, seeds data/species-photos.json
- Dropbox is a superset; clean-match → replace; unmatched → manual investigation (no auto-drop)
- Folder layout flat with encoded filenames `Genus species-{specimen}-{view}.{ext}` (same as Phase 13 photos)
- WebP (.webp[Q=80]) pinned in tile-config.json — pilot confirmed ~30% smaller than JPEG, OSD handles WebP DZI correctly
- DRY_RUN guard before BUNNY_API_KEY guard — enables pre-flight inspection without a real API key
- advanceStatus before deletion — status committed before tile files deleted (D-03 ordering invariant)
- Self-contained per-script helpers (redact, withRetry, logStage, walk) — no shared module; each script independently executable
- Phase 28 inserted as vertical-slice pilot before bulk phases to surface cross-phase integration risk; existing 28–31 renumbered to 29–32

**Issues Resolved:**
- Dropbox shared_link API does not return path_display — use '/' + entry.name as fallback; manifest backfilled
- downloadSharedFile marks 4xx (non-429) errors as err.retriable=false; withRetry bails immediately on these
- DROPBOX_TOKEN requires sharing.read + files.content.read scopes (not just files.metadata.read)
- species_slug lowercased unconditionally in tilePrefix (Phase 28 mixed-case lesson)

**Technical Debt Incurred:**
- Synonym curation mechanism in place but first curator pass not yet performed (Grammia, Eupithecia, Smerinthus ophthalmica residue outstanding)
- `*custom` Dropbox sub-folder deliberately deferred until contents understood
- WebP not yet active on bunny.net Optimizer (serving JPEG)
