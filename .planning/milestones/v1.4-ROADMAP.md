# Milestone v1.4: Image CDN

**Status:** ✅ SHIPPED 2026-04-22
**Phases:** 13–17
**Total Plans:** 13

## Overview

Migrated image storage from Git LFS to bunny.net object storage with CDN-native on-the-fly resizing. Removed build-time resize scripts and LFS from the repository history. Migrated the full legacy species dataset (1,348 species, 85,933 occurrence records) from a MySQL dump, replacing placeholder stub data.

## Milestone Summary

**Key Decisions:**

- D-01: CDN_BASE_URL is a hard-coded public constant in eleventy.config.js — no env var, no dotenv, no GitHub secret needed
- D-04: Species photo srcset deferred — pnwm-image-slideshow drops srcset on slotted img elements in connectedCallback
- D-05: CDN constant in web components is module-level, not a Lit property/attribute
- D-18: bunny.net Image Classes disabled (403 bug); direct Optimizer query params used instead
- LFS rewrite: clone from LOCAL repo when local commits are ahead of remote (prevents losing unpushed work)
- Migration: streaming readline for large SQL dumps; DB slug derivation for records.csv JOIN compatibility

**Issues Resolved:**

- Git LFS storage bloat (16,191 files across 356 commits) — purged via filter-repo
- Dead species photo copy block in copy-images.js causing ENOENT on fresh clones — removed
- Placeholder stub data (11 species, 667 records) replaced with full production dataset (1,348 species, 85,933 records)
- Django filenames with spaces rejected by build-data.js regex — widened to accept spaces

**Issues Deferred:**

- WebP conversion not yet active on bunny.net Optimizer (serving JPEG); enable in Pull Zone → Optimizer → WebP conversion toggle
- Species photo srcset — pnwm-image-slideshow web component architecture blocks it; deferred to v2
- Eleventy build time under 5 minutes (MAINT-03) — requires live CI observation
- GitHub LFS storage quota reclaim — requires repo deletion + recreation; accept billing

**Technical Debt Incurred:**

- WR-01: similar_species links may be silently dropped for record-only species (slug resolution gap in migrate-species.js:457-459)
- WR-02: safeSpecies sanitization logic duplicated in two loops (maintenance hazard)
- WR-03: Tests 3-7 in migrate-species.test.js have implicit ordering dependency on test 2
