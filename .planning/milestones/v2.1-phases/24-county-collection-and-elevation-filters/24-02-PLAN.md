## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → DOM | User interacts with `<select>` and `<input type="range">` controls inside the filter-bar shadow DOM. Values feed `_dispatchFilterChange()` and are echoed back into Lit-rendered text nodes (the elevation label) and `<option>` selectedness. |
| Parquet data → DOM | County and collection strings sourced from the species' Parquet file are interpolated into `<option>` value attributes and text content via Lit `html` template literals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-XS1 | Tampering / XSS via county/collection strings | `${this._counties.map(c => html`<option value=${c} ?selected=${this._county === c}>${c}</option>`)}` in `render()` | mitigate | Use Lit `html` tagged templates for ALL interpolation (no `innerHTML`, no string concat, no `unsafeHTML`). Lit auto-escapes attribute and text-node interpolations. Per UI-SPEC + research: "County/collection values displayed in dropdown options come from the Parquet data, not from user text input typed directly — no XSS risk from user-controlled strings entering the DOM" — but defense-in-depth via Lit's auto-escape applies regardless. Verified by acceptance criterion: no `innerHTML`, no `unsafeHTML` introduced. |
| T-24-XS2 | Tampering / XSS via slider numeric value | `_onElevationMinChange` / `_onElevationMaxChange` | mitigate | `Number(e.target.value)` coerces input to a number; non-numeric input yields `NaN`, and `Math.min(NaN, x)` returns `NaN` — the resulting reactive state would render `NaN` as text, not execute. HTML attribute `min="0" max="15000" step="100"` constrains the input element's accepted range at the platform level. No string interpolation into JS context. |
| T-24-PP | Tampering / Prototype pollution via filter state object | `_dispatchFilterChange()` `detail` literal | mitigate | The `detail` is built as a fresh object literal each dispatch, with only known string-keyed primitive properties. No `Object.assign(target, untrusted)`, no spread of attacker-controlled object into the detail. Consumer code in `species.njk` assigns `map.filters = e.detail` — does not mutate prototype. |
| T-24-EH | Tampering / Event handler injection | Lit `@change`/`@input` bindings | mitigate | All handlers are member methods of the Lit class, referenced by identifier (not by string). No `setAttribute('on...', ...)` patterns. Lit refuses to inline event handler strings. |
| T-24-SC | Supply chain | npm/pip/cargo installs | accept | No new packages introduced in this phase (RESEARCH.md Package Legitimacy Audit: "No new packages are installed in this phase"). Existing `lit` and `hyparquet` already in package.json. |
</threat_model>
