## Component Inventory

Components modified in this phase (for planner/executor reference):

### Modified: `pnwm-filter-bar.js` — county dropdown

- **Element:** `<div class="filter-group">` containing `<label>` + `<select>`
- **Label text:** `County` (visible, non-sr-only — matches "State" and "Record type" label pattern)
- **Default option:** `<option value="all">All counties</option>` (source: CONTEXT.md Claude's Discretion)
- **Options:** Dynamically populated from distinct non-null `county` values in Parquet data, alphabetically sorted (D-04, D-06)
- **Touch target:** `min-height: 44px` (existing rule covers all `select` elements)
- **ID:** `filter-county-${this.slug}` (matches existing `filter-state-${this.slug}` naming pattern)
- **Position in render():** After the existing "Record type" filter group, before the year range group

### Modified: `pnwm-filter-bar.js` — collection dropdown

- **Element:** `<div class="filter-group">` containing `<label>` + `<select>`
- **Label text:** `Collection` (visible, non-sr-only)
- **Default option:** `<option value="all">All collections</option>` (source: CONTEXT.md Claude's Discretion)
- **Options:** Dynamically populated from distinct non-null `collection` values in Parquet data, alphabetically sorted (D-04, D-06)
- **Touch target:** `min-height: 44px`
- **ID:** `filter-collection-${this.slug}`
- **Position in render():** After the county dropdown, before the year range group

### Modified: `pnwm-filter-bar.js` — elevation range slider

- **Element:** `<div class="filter-group year-range">` (reuse existing class) containing:
  - Visible `<label>`: `Elevation: ${this._elevationMin} – ${this._elevationMax} ft` (matches year range label pattern exactly, D-03)
  - `<div class="year-range-inputs">` containing two `<input type="range">`:
    - Min handle: `min="0"`, `max="15000"`, `step="100"`, `.value=${String(this._elevationMin)}`
    - Max handle: `min="0"`, `max="15000"`, `step="100"`, `.value=${String(this._elevationMax)}`
  - Each handle wrapped in a sr-only `<label>` (CONTEXT.md Claude's Discretion — match year range `class="sr-only"` pattern)
- **SR-only label text — min handle:** `Minimum elevation in feet`
- **SR-only label text — max handle:** `Maximum elevation in feet`
- **IDs:** `filter-elevation-min-${this.slug}`, `filter-elevation-max-${this.slug}`
- **Touch target:** `min-height: 44px` (existing rule covers all `input[type="range"]`)
- **Step size:** 100 ft (source: CONTEXT.md Claude's Discretion — "suggest 100 ft given the 0–15,000 range")
- **Position in render():** After the year range group, before the "Clear filters" link
- **CSS class reuse:** Use existing `.year-range` and `.year-range-inputs` classes — no new CSS classes needed

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Select county dropdown | `_onCountyChange(e)`: set `this._county = e.target.value`; call `_dispatchFilterChange()` |
| Select "All counties" | `_county` set to `'all'`; filter condition skipped in `filterRecords()` (D-08) |
| Select collection dropdown | `_onCollectionChange(e)`: set `this._collection = e.target.value`; call `_dispatchFilterChange()` |
| Select "All collections" | `_collection` set to `'all'`; filter condition skipped |
| Drag elevation min handle | `_onElevationMinChange(e)`: `this._elevationMin = Math.min(Number(e.target.value), this._elevationMax)`; dispatch |
| Drag elevation max handle | `_onElevationMaxChange(e)`: `this._elevationMax = Math.max(Number(e.target.value), this._elevationMin)`; dispatch |
| Handles reach same value | Min and max clamp to equal — both displays show same number; filtering produces exact-elevation match |
| Click "Clear filters" | Resets `_county = 'all'`, `_collection = 'all'`, `_elevationMin = 0`, `_elevationMax = 15000` alongside existing resets (D-02) |
| Any filter change | `pnwm-filter-change` event fires with full detail object; map and chart respond in real time (FILT-04) |
| Parquet load error | Dropdowns render with only the default "All" option; elevation slider renders at default bounds; no error UI shown to user |
| Species with no county/collection data | Dropdowns render with only the default "All" option — silently empty (D-04) |
