## Component Inventory

Components introduced or modified in this phase (for planner/executor reference):

### Modified: `.thumbnail-strip`
- **Container:** `overflow-x: auto`, `scroll-behavior: smooth`, `display: flex`, `gap: 4px`
- **Width:** 100% of the slideshow container
- **Height:** constrained to thumbnail height (93px) — no wrapping
- **Scrollbar:** hidden (`::-webkit-scrollbar { display: none }`, `scrollbar-width: none`) for clean appearance
- **ARIA:** `role="tablist"` with `aria-label="Photo thumbnails"`

### Modified: `.thumbnail` (each thumbnail image button)
- **Element:** `<button>` wrapping `<img>`
- **Height:** 93px fixed
- **Width:** natural aspect ratio (no square cropping; `width: auto`)
- **Border:** inactive = `2px solid transparent`; active = `2px solid var(--pico-primary)`
- **Cursor:** pointer
- **ARIA:** `role="tab"`, `aria-selected="true/false"`, `aria-label="Photo N of M: {alt text}"`
- **Focus ring:** browser default (`outline: revert`) — do not suppress

### Modified: `.controls` buttons (‹ / ›) — repurposed as strip scroll
- **Behavior (D-06):** Scrolls thumbnail strip left/right; does NOT change `_currentIndex`
- **Visibility (D-07):** Hidden (`display: none`) when strip does not overflow; visible only when `_stripOverflows === true`
- **Scroll amount per click:** half the strip container's `clientWidth`
- **Animation:** `scroll-behavior: smooth` on the strip container
- **Touch targets:** 44×44px minimum (existing constraint preserved)
- **ARIA:** `aria-label="Scroll thumbnails left"` / `aria-label="Scroll thumbnails right"`

### Removed: `.dots` and `.dot` elements
- Removed entirely per D-04. No replacement navigation chrome.

### Removed: `.index-label`
- Removed per D-05. Thumbnails make it redundant.

### Bug fix: `.lightbox-close` button (PHOTO-03)
- **Symptom to investigate:** Close button (`position: absolute` inside `position: fixed .lightbox`) may be unreachable due to shadow DOM stacking context or z-index layering. Inspect `_closeLightbox()`, the lightbox template, and shadow DOM rendering.
- **Contract:** Button must be reachable by pointer click and keyboard (Escape key already wired). After fix, clicking ✕ must call `_closeLightbox()` and remove `inert` from `<main>`.
- **No visual change expected** — the button appearance (white ✕, 44×44px, `top: 16px; right: 16px`) is already correct.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click thumbnail | Set `_currentIndex` to that thumbnail's index; update main image |
| Click ‹ button | Scroll strip left by `strip.clientWidth / 2` px; no index change |
| Click › button | Scroll strip right by `strip.clientWidth / 2` px; no index change |
| Overflow state change | ResizeObserver or `scrollWidth > clientWidth` check sets `_stripOverflows`; buttons show/hide reactively |
| Click main image | Open lightbox (`_openLightbox()`) — existing behavior unchanged |
| Click lightbox backdrop | Close lightbox — existing behavior unchanged |
| Click ✕ button | Close lightbox — bug fix target (PHOTO-03) |
| Press Escape | Close lightbox — existing behavior unchanged |
| Active thumbnail | Scrolled into view automatically when `_currentIndex` changes (use `scrollIntoView({ inline: 'nearest', behavior: 'smooth' })`) |
