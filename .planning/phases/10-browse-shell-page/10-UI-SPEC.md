## HTML Structure Contract

This phase is a Nunjucks template rewrite, not a component library phase. The structure contract is more important than abstract design tokens.

### `src/browse/index.njk` — required structure

```
---
layout: base.njk
title: Browse — PNW Moths
permalink: /browse/index.html
---

<h1>Browse all species</h1>

<script type="application/json" id="taxon-data" data-pagefind-ignore>
  {# serialized taxon JSON #}
</script>

<pnwm-taxon-browser></pnwm-taxon-browser>

<noscript>
  {# 4-level static listing: Family → Subfamily → Genus → Species #}
</noscript>
```

**Constraints (each is a hard requirement, not a preference):**

1. `<script type="application/json" id="taxon-data">` — not a `data-` attribute on `<pnwm-taxon-browser>`. Source: CONTEXT.md D-01.
2. `data-pagefind-ignore` on the script element — prevents Pagefind from indexing raw JSON. Source: ROADMAP.md Phase 12 SC-4 and STATE.md blockers.
3. `<pnwm-taxon-browser>` has no attributes. It reads taxonomy from the sibling script element at runtime. Source: CONTEXT.md D-01.
4. `<noscript>` block contains the complete 4-level hierarchy using the same `taxon` data variable. Source: CONTEXT.md D-04.
5. Species links in noscript use `/species/{slug}/` pattern. Source: CONTEXT.md D-05.
6. Subfamilies with `name === null` are flattened: genera appear directly under the family heading with no intermediate subfamily heading. Source: CONTEXT.md D-06.
7. JSON serialization: use Nunjucks `| tojson` or `| dump` filter. Source: CONTEXT.md code_context.
8. URL generation: use `{{ ('/species/' + sp.slug + '/') | url }}` pattern. Source: CONTEXT.md established patterns.

### Noscript listing — semantic HTML

The noscript block must render as navigable semantic HTML without CSS:

```
<h2>{Family name}</h2>
  <h3>{Subfamily name}</h3>   ← omit if subfamily.name is null
    <h4>{Genus name}</h4>
      <ul>
        <li><a href="/species/{slug}/">{genus} {species} — {common_name}</a></li>
      </ul>
```

Heading levels must be strictly nested (h2 → h3 → h4 → ul/li) for screen reader landmark navigation.

## Phase 10 Interaction Contract

Phase 10 has no interactive elements beyond standard anchor links. The interaction contract is entirely about the JS-off path:

| State | Behavior |
|-------|----------|
| JavaScript enabled | `<pnwm-taxon-browser>` element is present in DOM; renders nothing (unregistered custom element); Phase 11 will register it |
| JavaScript disabled | `<noscript>` block renders full 4-level taxonomy listing as semantic heading/list HTML |
| Pagefind indexing | `data-pagefind-ignore` on the script element prevents JSON content from appearing in search results |

No hover states, focus rings, or active states beyond Pico CSS defaults apply to Phase 10 content. The noscript listing inherits link styles from `--pico-primary` (olive green `#a4ab78`).
