**Step C — Rewrite `src/browse/index.njk`:**

Write the file with exactly this structure (per UI-SPEC.md HTML Structure Contract and
CONTEXT.md decisions D-01 through D-09):

```nunjucks
---
layout: base.njk
title: Browse — PNW Moths
permalink: /browse/index.html
---
<h1>Browse all species</h1>

<script type="application/json" id="taxon-data" data-pagefind-ignore>
  {{ taxon | tojson }}
</script>

<pnwm-taxon-browser></pnwm-taxon-browser>

<noscript data-pagefind-ignore>
  {% for family in taxon %}
    <h2>{{ family.name }}</h2>
    {% for subfam in family.subfamilies %}
      {% if subfam.name %}
        <h3>{{ subfam.name }}</h3>
      {% endif %}
      {% for genus in subfam.genera %}
        <h4>{{ genus.name }}</h4>
        <ul>
          {% for sp in genus.species %}
            <li><a href="{{ ('/species/' + sp.slug + '/') | url }}">
              <em>{{ genus.name }} {{ sp.name }}</em>{% if sp.common_name %} — {{ sp.common_name }}{% endif %}
            </a></li>
          {% endfor %}
        </ul>
      {% endfor %}
    {% endfor %}
  {% endfor %}
</noscript>
```

**Critical constraints (each is non-negotiable):**
1. JSON embed: `<script type="application/json" id="taxon-data" data-pagefind-ignore>` — NOT a `data-` attribute on `<pnwm-taxon-browser>` (per D-01)
2. `data-pagefind-ignore` on the script element AND on the noscript element
3. `<pnwm-taxon-browser></pnwm-taxon-browser>` — no attributes (per D-01)
4. When `subfam.name` is falsy (null), skip the `<h3>` entirely — genera appear directly under the family `<h2>` (per D-06)
5. Species links use `('/species/' + sp.slug + '/') | url` pattern (per D-05)
6. `sp.name` is the species epithet (not full binomial); use `genus.name + ' ' + sp.name` for the full binomial (per taxon.js data shape: `species: [{ slug, name, common_name }]`)
7. Use `tojson` filter for serialization — not `dump` (per CONTEXT.md code_context which lists `tojson` as confirmed available)
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSV data → build pipeline | `data/species.csv` and `data/images.csv` are trusted maintainer files; DuckDB reads them at build time |
| Build output → browser | `_site/browse/index.html` is static HTML served to untrusted browsers; no server-side processing at request time |
| taxon JSON → Pagefind indexer | Pagefind crawls the built site and can index `<script>` content if not explicitly excluded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | `<script type="application/json" id="taxon-data">` | mitigate | Add `data-pagefind-ignore` attribute on the script element to prevent Pagefind from indexing raw JSON taxonomy data (ASVS L1: avoid leaking internal data structures into search indexes) |
| T-10-02 | Information Disclosure | `<noscript>` static listing | mitigate | Add `data-pagefind-ignore` on the `<noscript>` element to prevent Pagefind from indexing the entire species listing as a single document and polluting search results |
| T-10-03 | Tampering | `{{ taxon | tojson }}` template expression | accept | `taxon` data originates from trusted maintainer-controlled CSV files processed at build time (no user input path); `tojson` filter escapes HTML-special characters, preventing script injection in the serialized JSON |
| T-10-04 | Denial of Service | Large taxon JSON payload | accept | Current dataset is ~700 species × 4-level tree; JSON payload is bounded at ~20–50 KB (well within browser parsing limits); no runtime query or unbounded growth path |
| T-10-05 | Spoofing | Unregistered `<pnwm-taxon-browser>` element | accept | The custom element is unregistered in Phase 10 output (Phase 11 adds registration); browsers render unregistered custom elements as empty inline elements with no security surface |
</threat_model>
