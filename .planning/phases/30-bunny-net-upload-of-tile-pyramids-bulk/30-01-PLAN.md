<objective>
Build `scripts/upload-tiles.js` — a manifest-driven Node.js CLI that uploads each `status: tiled` row's DZI tile directory to bunny.net Storage Zone `pnwmoths` at `species-tiles/{slug-lowercase}/{specimen_id}-{view}/`, advances the manifest row to `status: uploaded`, immediately deletes the local tile directory, and provides a pre-flight on-disk footprint summary before the first upload. Companion test file and `package.json` wiring ship in the same plan.

Purpose: Satisfies UPLOAD-01 (HTTP PUT pattern), UPLOAD-02 (manifest idempotency), and UPLOAD-03 (storage footprint sanity check). All execution happens on this local laptop — `var/tiles` is the default tile output dir per `scripts/tile-config.json`. No remote-server-specific paths anywhere in the script.

Output: Three deliverables — the script, its unit test file, and a `package.json` edit adding the `photos:upload` alias and the test file to the `npm test` glob.
</objective>
