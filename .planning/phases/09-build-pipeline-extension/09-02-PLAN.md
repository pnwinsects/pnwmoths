Sentinel key rule (from 09-RESEARCH.md Pitfall 4): internally use `'__none__'` as the map key for null-subfamily rows; expose `name: null` (not `name: '__none__'`) in the returned tree.

Two-query strategy (from 09-RESEARCH.md Pitfall 5): run species and images as TWO separate `runAndReadAll` calls (not a JOIN). Close connection AFTER both calls. Merge in JavaScript: build `bySpeciesSlug` image map, then walk the species tree to assign navImages per genus.

navImages selection per genus:
1. Filter `bySpeciesSlug[slug]` images for all species in the genus
2. Sort: `navigational === 'true'` first (descending), then `weight` ascending
3. Deduplicate by filename (a filename should not appear twice)
4. Slice to first 4

navImages roll-up:
- Genus level: up to 4 images from that genus's species (as above)
- Subfamily level: up to 4 images from the first genera in the subfamily (take the first image from each genus until 4 total)
- Family level: up to 4 images from the first subfamilies in the family (take the first image from each genus across subfamilies until 4 total)

       function pickNavImages(speciesSlugs, bySpeciesSlug) {
         const seen = new Set();
         const candidates = [];
         for (const slug of speciesSlugs) {
           for (const img of (bySpeciesSlug[slug] || [])) {
             if (!seen.has(img.filename)) {
               seen.add(img.filename);
               candidates.push({ ...img, species_slug: slug });
             }
           }
         }
         candidates.sort((a, b) => {
           const navA = a.navigational === 'true' ? 0 : 1;
           const navB = b.navigational === 'true' ? 0 : 1;
           if (navA !== navB) return navA - navB;
           return (a.weight ?? 999) - (b.weight ?? 999);
         });
         return candidates.slice(0, 4);
       }
