## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local CSV manifest → script | Operator-controlled file; only operator-edited content traverses this boundary |
| script → committed JSON | Script writes a committed artifact; downstream consumers (Eleventy + templates) trust it implicitly |
| committed JSON → Eleventy templates | Templates render `tiles_path` directly into HTML attributes (`high-res-specimens` JSON-escaped) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering | malformed CSV row (e.g., specimen_id containing `..` or `/`) | mitigate | toTilesPath uses template literal interpolation — no path resolution; downstream Nunjucks `tojson | escape` filter in species.njk line 42 escapes the JSON value into an HTML attribute, neutralizing injection. Tests cover the canonical happy-path values; out-of-band values would render as opaque strings that simply fail to resolve on the CDN. |
| T-31-02 | Information Disclosure | logStage prints content_hash prefix (12 chars) | accept | Content hashes are not secret; they are a SHA of public file contents. Established pattern across all photos:* scripts (Phase 26 D-13). |
| T-31-03 | Denial of Service | extremely large manifest (~5,000 rows) processed in one pass | accept | All operations are O(n) and operate on small JSON objects (specimen_id, view, tiles_path strings). Node easily handles 5,000 rows in <100ms. |
| T-31-04 | Tampering | committed JSON edited by hand to inject crafted tiles_path | accept | Script is the source of truth; any out-of-band edit is overwritten on next `photos:materialize` run. Same trust model as `data/plates.json`. |
| T-31-SC | Tampering | npm install of new packages | mitigate | No new packages installed (RESEARCH.md confirms zero new dependencies); package legitimacy gate not triggered. |
