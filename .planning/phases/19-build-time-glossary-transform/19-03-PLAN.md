**Change 3 — Load glossary.csv and build termMap at module scope.**

Add after the `const CDN_BASE_URL = "https://pnwmoths.b-cdn.net";` line (before

```javascript
// Load glossary terms once at startup. termMap is sorted longest-first and
// has pre-compiled regexes — shared across all addTransform invocations via closure.
// csv-parse/sync is synchronous; no async needed here.
const glossaryRows = parseCsv(readFileSync("data/glossary.csv"), {
  columns: true,
  skip_empty_lines: true,
});
const termMap = buildTermMap(glossaryRows, CDN_BASE_URL);
```

**Change 4 — Register the addTransform inside export default function.**

Add inside `export default function (eleventyConfig) { ... }`, after the last
`eleventyConfig.addFilter(...)` call and before the `eleventyConfig.addGlobalData(...)` call.
Use a regular `function` callback (NOT an arrow function) to preserve `this.page.outputPath`
binding — arrow functions do not have their own `this` and will break the outputPath guard:

```javascript
  // Annotate species prose pages at build time: wrap first occurrences of glossary
  // terms in <abbr class="glossary-term"> elements.
  // Guard 1: skip non-HTML outputs (outputPath is false for permalink:false pages)
  // Guard 2: skip non-species pages (glossary, browse, home, etc.)
  eleventyConfig.addTransform("glossary-terms", function (content) {
    const outputPath = this.page.outputPath;
    if (!outputPath || !outputPath.endsWith(".html")) return content;
    if (!outputPath.includes("/species/")) return content;
    return applyGlossaryTerms(content, termMap);
  });
```

**Critical notes for implementation:**
- Do NOT use an arrow function for the addTransform callback — `this.page.outputPath`
  is `undefined` in arrow functions.
- Do NOT add a duplicate `import ... from "node:fs"` line — extend the existing one.
- Do NOT hardcode "/pnwmoths/" anywhere (per project MEMORY.md constraint — the
  pathPrefix conditional already handles this correctly).
- The `termMap` variable is defined at module scope (Change 3) and is therefore
  accessible inside the addTransform callback via closure.
