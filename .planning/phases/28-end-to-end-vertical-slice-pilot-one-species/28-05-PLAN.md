Hand-edited JSON schema for `data/species-photos.json` (from RESEARCH.md Pattern 2):

```json
{
  "{pilot-species-slug}": {
    "high_res_available": true,
    "specimens": [
      { "specimen_id": "A", "view": "D", "tiles_path": "species-tiles/{slug}/A-D" },
      { "specimen_id": "A", "view": "V", "tiles_path": "species-tiles/{slug}/A-V" }
    ]
  }
}
```

Field rules:
- Key is the species slug (matches `sp.slug` in `data/species.csv` / species.njk).
- `specimens[].specimen_id` — exact string from the manifest's `specimen_id` column (`A`, `B`, or institutional accession like `WWUC0000003275`).
- `specimens[].view` — `'D'` or `'V'` (uppercase).
- `specimens[].tiles_path` — RELATIVE path to the tile directory (no leading slash, no host). The full DZI URL OSD fetches is `{cdnBaseUrl}/{tiles_path}/{specimen_id}-{view}.dzi`.

Browser verification target URLs (from Plan 03 verify block):
- Lightbox-triggered .dzi fetch: `https://pnwmoths.b-cdn.net/species-tiles/{slug}/{specimen_id}-{view}/{specimen_id}-{view}.dzi`
- Tile fetches OSD derives: `https://pnwmoths.b-cdn.net/species-tiles/{slug}/{specimen_id}-{view}/{specimen_id}-{view}_files/{level}/{col}_{row}.jpg`

Dev preview command (project standard, per package.json line 21):
- `npm run dev` runs `npm run build:data && eleventy --serve` (typically on `http://localhost:8080`)
