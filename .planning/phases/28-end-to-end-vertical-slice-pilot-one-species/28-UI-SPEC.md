# Phase 28 — UI Design Contract

> Visual and interaction contract for the end-to-end vertical-slice pilot.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope constraint:** This is a one-species pilot. Every decision here is optimized for
> correctness and no-regression safety, not novelty. The viewer wiring will be generalized
> in Phase 32; this contract governs only the pilot branch.

## Spacing Scale

Declared values (must be multiples of 4). Source: `theme.css` explicit values + Pico CSS base.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Thumbnail gap, icon gap |
| sm | 8px | Control gap, caption margin, thumbnail strip gap |
| md | 16px | Lightbox close button inset from edge, section padding |
| lg | 24px | Content wrapper padding, nav padding |
| xl | 32px | Layout section gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

### Grid-aligned exceptions

These values are multiples of 4 and are part of the spacing system:

- Touch targets: minimum 44×44px — already established in `pnwm-image-slideshow.js` (`.controls button { min-width: 44px; min-height: 44px; }`). The OSD lightbox close button is 44×44px (existing contract; do not reduce). 44 is a multiple of 4.
- OSD viewer height: 70vh (min 400px) — matches the existing `pnwm-plate-viewer` contract; use the same value for the species lightbox OSD container.

### Frozen legacy dimensions (outside the 4px grid rule)

These values are not spacing tokens and must not be treated as such. They exist solely to preserve Phase 23 contracts and are frozen.

- Thumbnail height: **93px** — established in Phase 23; not a spacing token; not a multiple of 4 (93 / 4 = 23.25); frozen by prior phase contract; do not change; do not apply the 4px grid rule to this value.

## Typography

Source: `theme.css` + Pico CSS defaults.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem Pico default) | 400 | 1.5 |
| Label / caption | 14px (0.875rem) | 400 | 1.4 |
| Heading (h1–h6) | Pico scale (h1 ≈ 2rem, h2 ≈ 1.75rem, etc.) | 700 | 1.2 |

The specimen metadata bar (specimen_id + view) rendered inside the OSD lightbox and all other caption-style text both use **14px / 0.875rem** (weight 400). This consolidates the former `0.8rem (~13px)` caption size and the `14px` specimen-metadata size — a one-pixel difference that produces no hierarchy benefit. The existing `.caption-line` CSS uses `font-size: 0.8rem`; the executor must update that rule to `font-size: 0.875rem` as part of this phase to align with the 14px token. If that change conflicts with existing CSS specificity, document the conflict in PILOT-LESSONS.md and revert to `0.8rem` only for the affected rule — but do not introduce a third size.

No new font weights are introduced in this phase. Two weights in use: 400 (body, captions) and 700 (headings).

## Interaction Contract

### OSD Viewer — Lightbox States

| State | What the user sees |
|-------|--------------------|
| Lightbox closed | Normal thumbnail strip carousel (Phase 23 — unchanged) |
| Lightbox opening (high-res) | Lightbox overlay fades in; OSD `<div>` appears with dark background; OSD controls (home, zoom in, zoom out) appear after async import completes |
| OSD loading tiles | OSD built-in loading state (spinner / blank tiles progressively fill in) — no custom loading indicator required for pilot |
| OSD loaded | Full-res tiles visible at fit-to-screen zoom; navigator minimap shown in corner (pilot config: `showNavigator: true`); specimen metadata bar beneath canvas |
| Error — `.dzi` fetch fails | OSD shows blank canvas; specimen metadata bar still visible; no crash; console error logged |
| Error — CORS on `.dzi` | Same as above; error surfaced in browser console; pilot will check this explicitly |
| Lightbox closing | Close button (×) at top-right (44×44px, existing contract); Escape key; click outside OSD canvas on overlay; OSD `.destroy()` called on close to prevent WebGL/listener leaks |

The OSD canvas is the primary focal point within the lightbox overlay; all other elements (metadata bar, close button, navigator minimap) are secondary and must not visually compete with the canvas. OSD built-in icon controls (home, zoom in, zoom out) provide their own `title` attributes per the OSD library — this is the intended accessibility path for those controls; do not add redundant custom `aria-label` attributes to the OSD-managed buttons.

### OSD Configuration (pilot defaults)

```
prefixUrl: '/osd-images/'          -- existing contract; nav button PNGs
tileSources: {dziUrl}              -- full CDN URL to .dzi descriptor
visibilityRatio: 1.0
minZoomLevel: 0.5
defaultZoomLevel: 0                -- fit to viewport on open
showNavigator: true                -- pilot setting; record in PILOT-LESSONS.md whether useful
showRotationControl: false         -- consistent with plate viewer
```

The `dziUrl` is constructed in the Lit component from the `tiles_path` attribute:
`${cdnBaseUrl}/${tilesPath}/${specimenId}-${view}.dzi`

where `cdnBaseUrl` is passed as an attribute from the Nunjucks template.

### OSD Lifecycle

- OSD instance is created **after** `await this.updateComplete` following `_lightboxOpen = true`.
  Reason: the lightbox `<div>` does not exist in shadow DOM until after Lit renders.
- OSD instance is destroyed on lightbox close via `this._osdViewer.destroy(); this._osdViewer = null;`
  Reason: prevents leaked event listeners across multiple open/close cycles.
- OSD is NOT initialized in `firstUpdated()` — this is the primary pitfall from RESEARCH.md.

### Specimen Metadata Bar

Rendered below the OSD canvas, inside the lightbox overlay:

```
Specimen {specimen_id} · {view label}
```

where `view label` is "Dorsal" when `view === 'D'` and "Ventral" when `view === 'V'`.

Style: `.caption-line` (existing class) — `font-size: 0.875rem; color: var(--pico-muted-color); text-align: center`. See Typography section for the consolidation from `0.8rem` to `0.875rem`.

When a species has multiple specimens, the metadata bar updates to reflect the currently displayed specimen. Pilot species has 1–3 specimens; the carousel thumbnail strip (unchanged) controls which specimen the OSD opens to.

### Mobile Behavior

- OSD viewer `height: 70vh; min-height: 400px` — matches plate viewer; usable on 375px wide phones.
- OSD touch gestures (pinch-to-zoom, drag-to-pan) work out of the box via OSD's built-in touch handling — no custom touch code.
- Close button remains at 44×44px minimum (existing constraint).
- Lightbox overlay covers full viewport (`position: fixed; inset: 0`) — existing behavior.

## Copywriting Contract

Source: REQUIREMENTS.md (PILOT-01), RESEARCH.md (Pattern 5), phase summary.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — no explicit CTA button is added; the existing thumbnail/image click already opens the lightbox |
| Lightbox close button | `aria-label="Close lightbox"` (existing); visible glyph: `×` (U+00D7) |

| Loading state | No copy — OSD built-in progressive tile load (blank-to-full); no spinner text added for pilot |
| Error state (OSD blank) | No copy added for pilot — CORS and 404 errors surface in browser DevTools only; user sees blank canvas. Record in PILOT-LESSONS.md whether an error state copy is needed before Phase 32. |

| Destructive actions | None in this phase |

Note on error state: deliberately minimal for the pilot. If OSD fails to load tiles, the user sees a dark blank canvas. This is acceptable pilot behavior — the goal is to surface CORS/URL issues to the operator, not to ship a polished error UX. Phase 32 will address error copy when generalizing.

## No-Regression Design Constraints

These constraints are first-class design requirements, not implementation notes:

1. **Phase 23 lightbox unchanged for non-high-res species.** The OSD code path is gated behind `highResAvailable === true`. Every species page without a `species-photos.json` entry renders identically to today.
2. **Thumbnail strip unchanged.** The carousel (thumbnail click → main image swap) operates in the same layer as before. OSD only occupies the lightbox layer.
3. **Existing `.lightbox` CSS class unchanged.** The OSD viewer is placed inside a new `<div id="osd-viewer">` inside the existing lightbox structure — not replacing it.
4. **OSD import is dynamic.** `import('openseadragon')` is already code-split by Vite for the plate viewer. The slideshow component must also use dynamic import so OSD does not load on species pages that never open the lightbox.
5. **Build page count must remain 1,364.** The pilot adds one JSON data file and one Eleventy data file — no page generation changes.
