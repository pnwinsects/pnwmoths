<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time JSON -> Nunjucks attribute | `speciesPhotos[sp.slug]` data is operator-authored, committed; rendered into HTML attribute via `| tojson | escape` |
| HTML attribute -> Lit component shadow DOM | `high-res-specimens` JSON string -> JSON.parse in connectedCallback with try/catch + `[]` fallback |
| Component state -> OpenSeadragon | `_buildDziUrl` constructs a URL string from operator-authored fields; passed as `tileSources`; OSD treats as a URL (does not execute) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-04-01 | Tampering / XSS | High-res-specimens JSON attribute injection | mitigate | Template uses `| tojson | escape` per PATTERNS.md; JSON.parse in connectedCallback is in a try/catch that logs and falls back to `[]` (per RESEARCH.md §Security Domain V5); OSD `tileSources` as URL string does not execute script |
| T-28-04-02 | Denial of Service | OSD initialization race on `updateComplete` | mitigate | `await this.updateComplete` before `shadowRoot.querySelector('#osd-viewer')` per RESEARCH.md Pitfall 5; without the await OSD fails silently — failure is non-fatal (lightbox stays open, OSD does not attach) |
| T-28-04-03 | Resource leak | WebGL contexts / event listeners on lightbox reopen | mitigate | `_closeLightbox` calls `this._osdViewer?.destroy(); this._osdViewer = null;` per RESEARCH.md Pattern 6 |
| T-28-04-04 | Regression | OSD branch accidentally activated for non-high-res species | mitigate | Render branch gates on BOTH `this.highResAvailable === true` AND `this._highResSpecimens?.length > 0` per UI-SPEC §No-Regression Design Constraints + RESEARCH.md Pitfall 6; verified at build time by the zero-`high-res-available`-attributes check in Task 2 acceptance |
| T-28-04-05 | Information Disclosure | Stack trace leaks via console.error on bad JSON | accept | Logging to browser console; same trust model as existing `console.warn` in `_data/plates.js`; no sensitive data in stack trace |

Block-severity threshold: high. T-28-04-04 (regression) is the highest-priority risk because it would break every existing species page — mitigated by the dual-condition gate and verified by the zero-attribute build smoke check.
