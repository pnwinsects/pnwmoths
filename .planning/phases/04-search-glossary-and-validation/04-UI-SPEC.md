## Typography

Pico CSS defaults. No overrides. Same as Phases 2 and 3.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / metadata | 14px | 400 (regular) | 1.4 |
| Heading (h2 — section) | 20px | 700 (bold) | 1.2 |
| Heading (h1 — page title) | 28px | 700 (bold) | 1.2 |

**Pagefind UI widget typography:** Pagefind renders its own result list. Override `--pagefind-ui-font` to match the Pico system-ui stack: `system-ui, "Segoe UI", Roboto, Helvetica, Arial, sans-serif`. Do not override font-size — Pagefind uses relative units that inherit correctly from Pico's base.

**Glossary page:**
- Each term: `<dt>` at 16px bold (700) — Pico `<dl>` default.
- Each definition: `<dd>` at 16px regular (400), line-height 1.5 — Pico `<dl>` default.
- Alphabetic section heading (`<h2>` for each letter): 20px bold (700).

**Source:** Phase 2 UI-SPEC (established). No changes in Phase 4.

## Color

Pico CSS custom properties. No hex values hardcoded. Same palette as Phases 2 and 3.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--pico-background-color)` | Page background for search and glossary pages |
| Secondary (30%) | `var(--pico-card-background-color)` + `var(--pico-muted-border-color)` | Glossary definition rows (alternating or bordered), search result item containers |
| Accent (10%) | `var(--pico-primary)` (#0172ad) | Search result title links, active Pagefind UI states, glossary term internal anchors |
| Destructive | not applicable | No destructive actions in Phase 4 |

Accent reserved for: search result title links rendered by Pagefind widget, glossary alphabetic jump-to-letter anchor links. Accent is NOT used for decorative elements, definition text, or page chrome.

**Pagefind CSS custom property overrides** (place in `<style>` block within `src/search/index.njk`):

```css
#search {
  --pagefind-ui-primary: var(--pico-primary);
  --pagefind-ui-text: var(--pico-color);
  --pagefind-ui-background: var(--pico-background-color);
  --pagefind-ui-border: var(--pico-muted-border-color);
  --pagefind-ui-tag: var(--pico-card-background-color);
  --pagefind-ui-font: system-ui, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
}
```

Do NOT set `--pagefind-ui-scale` — inherit Pagefind's default (1.0 at 16px base matches Pico).

**Source:** D-06 (CONTEXT.md) — "Style Pagefind UI via CSS custom properties to match Pico CSS palette."

## Component Inventory

### Search Page (`/search/`)

Replaces the Phase 2 stub. Full structure:

```html
<h1>Search</h1>
<link href="/pagefind/pagefind-ui.css" rel="stylesheet">
<style>
  #search {
    --pagefind-ui-primary: var(--pico-primary);
    --pagefind-ui-text: var(--pico-color);
    --pagefind-ui-background: var(--pico-background-color);
    --pagefind-ui-border: var(--pico-muted-border-color);
    --pagefind-ui-tag: var(--pico-card-background-color);
    --pagefind-ui-font: system-ui, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
  }
</style>
<div id="search"></div>
<script src="/pagefind/pagefind-ui.js"></script>
<script>
  window.addEventListener('DOMContentLoaded', () => {
    new PagefindUI({ element: '#search', showSubResults: false });
  });
</script>
```

- `showSubResults: false` — prevents sub-headings within a single page from appearing as separate results (cleaner UX for ~700 species pages).
- The Pagefind bundle is output to `_site/pagefind/` by `pagefind --site _site` post-build. Eleventy passthrough copy is not needed — the directory is generated directly into `_site/`.

### Glossary Page (`/glossary/`)

Replaces the Phase 2 stub. Rendered entirely at build time by Eleventy from `data/glossary.csv` via DuckDB. Structure:

```html
<h1>Glossary</h1>

<nav aria-label="Alphabetic index">
  <ul>
    <!-- one <li><a href="#letter-A">A</a></li> per letter that has at least one term -->
  </ul>
</nav>

<!-- repeated per letter group -->
<section id="letter-{A}">
  <h2>{A}</h2>
  <dl>
    <!-- repeated per term in this letter group -->
    <dt id="term-{slug}">{term}</dt>
    <dd>
      {definition}
      <!-- if image_filename is non-null: -->
      <figure>
        <img src="/images/glossary/{image_filename}"
             alt="{term}"
             width="120">
        <figcaption>{photographer}</figcaption>
      </figure>
    </dd>
  </dl>
</section>
```

- Alphabetic index `<nav>` renders only letters that have at least one term. Not all 26 letters may be present.
- Each `<dt>` receives an `id="term-{slug}"` where slug is the term lowercased with spaces replaced by hyphens, for deep-linking.
- Image is omitted entirely (no broken `<img>`) when `image_filename` is null/empty.
- Photographer credit is omitted when `photographer` is null/empty.
- DuckDB sorts terms `ORDER BY term ASC` at build time — no template-level sorting needed.

### Validation Scripts (no UI, build-only)

These produce no rendered HTML. They are Node.js scripts or CLI tools invoked in the `build` npm script. No UI contract applies.

## Interaction States

### Search Page

| State | Visual |
|-------|--------|
| Initial (no query) | Pagefind renders a search input box with placeholder text; no results shown |
| Typing | Pagefind shows live results as user types (built-in Pagefind behavior) |
| Results found | List of species page links with excerpt snippets |
| No results | Pagefind built-in no-results message (do not override copy — Pagefind handles it) |
| JS disabled | Fallback: "Search requires JavaScript. Browse all species instead." (see Copywriting) |

### Glossary Page

| State | Visual |
|-------|--------|
| Default | Full alphabetized term list; alphabetic jump index at top |
| Term with image | `<figure>` below definition, image at max-width 120px |
| Term without image | Definition only; no placeholder, no broken image |
| Build-time empty (no terms) | Build fails — glossary.csv must have at least one row |

## Accessibility

- Search page: Pagefind widget renders its own accessible markup. Ensure `<h1>Search</h1>` precedes the `<div id="search">` in DOM order.
- Glossary `<nav>` for alphabetic index: `aria-label="Alphabetic index"` on the nav element.
- Glossary `<dl>`: `<dt>` and `<dd>` pairs with id anchors for deep-linking.
- Glossary images: `alt="{term}"` (the term being defined, not a description of the photo).
- Glossary `<figcaption>`: photographer credit text — no aria attributes needed.
- Page `<title>` pattern: `Search — PNW Moths`, `Glossary — PNW Moths` (consistent with Phase 2 convention).

**Source:** Phase 2 accessibility contract (extended for new pages).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Search page `<h1>` | "Search" |
| Search JS-off fallback | "Search requires JavaScript. Browse all species instead." |
| Glossary page `<h1>` | "Glossary" |
| Glossary term with no definition (edge case) | *(omit the term — DuckDB query excludes null definitions at build time)* |
| Glossary term with no image | *(silently omit the `<figure>` — no placeholder copy needed)* |
| Pagefind no-results | *(use Pagefind default — do not override)* |
| Destructive confirmation | none — no destructive actions in Phase 4 |

**Primary CTA for Phase 4:** "Browse all species" — used in the JS-off fallback on the search page, linking to `/browse/`. This is the same CTA used throughout Phase 2 stubs.

**No custom error copy for validation failures** — validation scripts write to stdout/stderr at build time, not to rendered HTML.

**Source:** D-05, D-06 (CONTEXT.md); Phase 2 copywriting contract (browse CTA).

## Pagefind Indexing Contract

Pagefind exclusions are applied via `data-pagefind-ignore` attributes in the Eleventy templates (not in this phase — templates were established in phases 2 and 3, but the exclusion attributes must be added to them as part of Phase 4 implementation).

| Element | Action |
|---------|--------|
| `<nav>` (site-wide navigation) | Add `data-pagefind-ignore` |
| `<footer>` (if present) | Add `data-pagefind-ignore` |
| `<pnwm-filter-bar>`, `<pnwm-occurrence-map>`, `<pnwm-phenology-chart>` | Add `data-pagefind-ignore` (these render no static text, but belt-and-suspenders) |
| `<noscript>` blocks | Add `data-pagefind-ignore` |
| Species `<h1>`, `<dl>` (taxonomy), `<article>` (prose), `<section>` similar species | Indexed by default (no attribute needed) |

**Source:** D-08, D-09 (CONTEXT.md). Occurrence data exclusion is architecturally guaranteed (never in HTML).
