## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27.01-01 | Tampering | CSV-injection (spreadsheet auto-formula evaluation when a value starts with `=`, `+`, `-`, `@`) | accept | The header-only seed contains no user data. When the curator later adds rows, the values are lowercased binomials and species slugs — none of which can legitimately start with formula-trigger characters. The downstream consumer is `csv-parse/sync`, not a spreadsheet macro engine, so even an injected formula would be read as a plain string by Plan 02. The curator runbook (Plan 03) explicitly tells the curator never to paste arbitrary text into the file; values are always lowercase letters + hyphens + spaces. |
| T-27.01-02 | Tampering | Path traversal via `to_species_slug` containing `../` or absolute paths | accept | `to_species_slug` is never used as a filesystem path. Plan 02's loader resolves it against an in-memory `species.bySlug` Map (string-equality lookup); rows whose target slug is not in that Map are dropped and a `synonym-warn` line is logged. There is no filesystem `open()` keyed on this column anywhere in the codebase. |
| T-27.01-03 | Information disclosure | Curator accidentally commits sensitive data | accept | The file's purpose is to map taxonomic binomials. The curator runbook (Plan 03) calls out that values are always lowercase letters, hyphens, and spaces — no PII, no secrets. PR review at commit time catches anything weirder. |
| T-27.01-SC | Tampering | npm package legitimacy (no new packages) | accept | This plan adds zero npm dependencies. Verified by the absence of any `package.json` modification in `<files>`. |
