<task type="auto">
  <name>Task 1: Add subfamily column to species.csv</name>
  <files>data/species.csv</files>
  <read_first>
    - data/species.csv — read the full file; verify current header and all data rows before editing
  </read_first>
  <action>
    Append `subfamily` as the rightmost column in the header row and add a trailing comma (blank value) to every existing data row.

    Current header (line 1):
    ```
    id,genus,species,common_name,noc_id,authority,family,similar_species
    ```
    New header:
    ```
    id,genus,species,common_name,noc_id,authority,family,similar_species,subfamily
    ```

    Every existing data row ends with the similar_species value. Append a trailing comma and no value so each row has a blank subfamily cell. Example:
    ```
    1,Acronicta,americana,American Dagger Moth,9200,Harris 1841,Noctuidae,autographa-californica,
    2,Autographa,californica,Alfalfa Looper,8914,Speyer 1875,Noctuidae,acronicta-americana,
    ```

    Do NOT add any non-blank subfamily values — all existing rows have unknown subfamily. Blank cells are intentional; they will be coerced to NULL by DuckDB `nullstr = ''` in Plan 02 (D-03, D-04).
  </action>
  <verify>
    <automated>node -e "const fs=require('fs'); const lines=fs.readFileSync('data/species.csv','utf8').trim().split('\n'); const hdr=lines[0].split(','); if(!hdr.includes('subfamily')) throw new Error('no subfamily column'); const last=hdr.indexOf('subfamily'); const allBlank=lines.slice(1).every(l=>{ const cols=l.split(','); return cols[last]===''; }); if(!allBlank) throw new Error('some rows have non-blank subfamily'); console.log('OK: subfamily column present and all values blank');"</automated>
  </verify>
  <acceptance_criteria>
    - `data/species.csv` line 1 is exactly: `id,genus,species,common_name,noc_id,authority,family,similar_species,subfamily`
    - Every subsequent row ends with a trailing comma and no characters before the newline (blank subfamily value)
    - Row count (excluding header) is unchanged from before the edit
  </acceptance_criteria>
  <done>species.csv header contains subfamily as its 9th column; all data rows have a blank 9th field.</done>
</task>

<task type="auto">
  <name>Task 2: Add navigational column to images.csv</name>
  <files>data/images.csv</files>
  <read_first>
    - data/images.csv — read the full file; verify current header and all data rows before editing
  </read_first>
  <action>
    Append `navigational` as the rightmost column in the header row and add a trailing blank value to every existing data row.

    Current header (line 1):
    ```
    species_slug,filename,photographer,weight,license,view,specimen
    ```
    New header:
    ```
    species_slug,filename,photographer,weight,license,view,specimen,navigational
    ```

    Every existing data row ends with the specimen value (which is itself often blank). Append a trailing comma and no value so each row has a blank navigational cell. Example:
    ```
    acronicta-americana,01.jpg,Jane Doe,1,CC BY 4.0,,,
    acronicta-americana,02.jpg,John Smith,2,CC BY 4.0,,,
    ```

    Note: the existing `specimen` column is already often blank, so rows may already end with a trailing comma before this edit. After the edit every row must have exactly one MORE comma than before, placing a blank value at position 8 (0-indexed: index 7 = specimen, index 8 = navigational).

    Do NOT set any row to `true` — no existing images are marked navigational yet. Blank = NULL via `nullstr = ''` = treated as false by consuming code (D-01).
  </action>
  <verify>
    <automated>node -e "const fs=require('fs'); const lines=fs.readFileSync('data/images.csv','utf8').trim().split('\n'); const hdr=lines[0].split(','); if(!hdr.includes('navigational')) throw new Error('no navigational column'); if(hdr.indexOf('navigational')!==7) throw new Error('navigational not at index 7'); const allBlank=lines.slice(1).every(l=>{ const cols=l.split(','); return cols[7]===''; }); if(!allBlank) throw new Error('some rows have non-blank navigational'); console.log('OK: navigational column present and all values blank');"</automated>
  </verify>
  <acceptance_criteria>
    - `data/images.csv` line 1 is exactly: `species_slug,filename,photographer,weight,license,view,specimen,navigational`
    - Every subsequent row has exactly 8 comma-separated fields with the 8th (index 7) being blank
    - Row count (excluding header) is unchanged from before the edit
  </acceptance_criteria>
  <done>images.csv header contains navigational as its 8th column; all data rows have a blank 8th field.</done>
</task>
