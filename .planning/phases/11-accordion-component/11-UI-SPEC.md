# Phase 11 — UI Design Contract

> Visual and interaction contract for the Accordion Component (`<pnwm-taxon-browser>`). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Pico CSS classless with custom property overrides |
| Preset | not applicable |
| Component library | Pico CSS v2 (classless mode via `/css/pico.min.css`) |
| Icon library | none |
| Font | Open Sans (body), Spinnaker (headings) — Google Fonts |

**Source:** `src/styles/theme.css`, Phase 10 UI-SPEC baseline.

shadcn gate: not applicable — this project uses Eleventy + Lit web components, not React/Next.js/Vite.

**Light DOM mandate (D-09):** `<pnwm-taxon-browser>` uses `createRenderRoot() { return this; }`. Pico CSS element selectors (headings, `<select>`, `<a>`, `<img>`) must reach component internals. Shadow DOM is forbidden for this component.

**Inline styles policy (Pitfall 4):** Because light DOM has no style isolation, component-specific layout rules (image strip flex, muted opacity) use inline styles or tightly namespaced class names (`.pnwm-tb-*` prefix). No `<style>` block is injected into the component. No new Pico class names are introduced that could collide globally.

## Spacing Scale

Inherited from Phase 10 baseline. No new values introduced for Phase 11.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between nav thumbnail images in the image strip |
| sm | 8px | Gap between toolbar controls (checkbox + select); accordion row vertical padding |
| md | 16px | Toolbar horizontal padding; accordion row left indent per level |
| lg | 24px | Section padding within the content wrapper |
| xl | 32px | Vertical spacing between top-level family sections |
| 2xl | 48px | Not used in Phase 11 |
| 3xl | 64px | Not used in Phase 11 |

Exceptions:
- Image strip height: **93px** (user-specified minimum for specimen detail visibility; not a spacing token — it is a fixed dimension). Source: D-02, CONTEXT.md specifics.
- Touch targets: `<select>` and `<input type="checkbox">` must meet 44px minimum tap target height. Pico CSS applies `min-height: 44px` to form controls natively — no override needed.

**Source:** `src/styles/theme.css` — `.content-wrapper` uses `padding: 24px`; spacing scale from Phase 10 UI-SPEC.

## Typography

All typographic values inherited from the global contract. Phase 11 introduces no new typographic roles. The accordion renders semantic headings (h2 → h3 → h4) and body links — Pico CSS and `theme.css` style these automatically via light DOM.

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px (Pico default) | 400 (regular) | 1.5 | Open Sans, Verdana, sans-serif |
| Label / small | 14px | 400 (regular) | 1.5 | Open Sans, Verdana, sans-serif |
| Heading — family (h2) | 20px (Pico h2 default) | 400 (Spinnaker display weight) | 1.2 | Spinnaker, Open Sans, sans-serif |
| Heading — genus (h4) | ~16px (Pico h4 default) | 400 (Spinnaker display weight) | 1.2 | Spinnaker, Open Sans, sans-serif |

**Accordion heading levels follow the noscript block convention (Phase 10):**
- `<h2>` — Family
- `<h3>` — Subfamily (omitted when `subfamily.name === null`)
- `<h4>` — Genus
- Species: `<li><a>` links — not headings

**Source:** `src/styles/theme.css` — `h1–h6` font-family override; Pico classless default size scale.

## Color

All color values inherited from `theme.css`. Phase 11 introduces one additional semantic value: muted state for filtered-out taxa.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f3e8ba` (warm cream) | Page background (`--pico-background-color`) |
| Secondary (30%) | `#ffffff` (white) | Content wrapper background (`.content-wrapper`) |
| Accent (10%) | `#a4ab78` (olive green) | Nav link hover state; species factsheet links (inherits `--pico-primary`) |
| Header/footer | `#000000` (black) | Site header, footer, nav backgrounds |
| Text on dark | `#ffffff` | Nav links, footer text |
| Muted state | `opacity: 0.35` applied inline | Accordion rows whose taxon has no occurrence records in the selected state (D-06) |
| Destructive | not applicable | No destructive actions in Phase 11 |

Accent reserved for: nav link hover state only; species links (inheriting `--pico-primary`). The accordion headings and expand/collapse buttons use default Pico heading and button styles — not the accent color.

**Muted state detail (D-06):** When a state is selected and a taxon has no records in that state, apply `style="opacity:0.35"` inline to that taxon's row container. The taxon remains visible in the DOM — it is not hidden, removed, or `display:none`. This communicates "WA has no Lasiocampidae" as useful information to naturalists.

**Source:** `src/styles/theme.css`; D-06 from CONTEXT.md.

## Component Structure Contract

This phase implements a Lit web component, not a template rewrite. The structure contract defines the rendered DOM shape and interaction rules for `<pnwm-taxon-browser>`.

### Rendered DOM skeleton

```
<pnwm-taxon-browser>  ← light DOM root (createRenderRoot returns this)
  <div class="pnwm-tb-toolbar">
    <label>
      <input type="checkbox" checked>   ← show/hide images toggle (D-08, discretion)
      Show images
    </label>
    <label for="pnwm-tb-state-filter">Filter by state</label>
    <select id="pnwm-tb-state-filter" ?disabled=${!statesLoaded}>
      <option value="">All states</option>
      <option value="ID">ID</option>
      ...
    </select>
  </div>

  <!-- Repeat per family -->
  <div class="pnwm-tb-family-row" style="opacity:0.35 | ''">   ← muted if filtered
    <h2>
      <button type="button" aria-expanded="false|true">
        {Family name}
      </button>
    </h2>
    <!-- Image strip — visible when collapsed AND showImages is true (BROWSE-02) -->
    <div class="pnwm-tb-strip" style="display:inline-flex;gap:4px;overflow-x:auto">
      <img src="/images/{filename}" alt="" loading="lazy"
           style="height:93px;width:auto;object-fit:cover;flex-shrink:0">
      ...
    </div>
    <!-- Subfamily list — visible only when family is expanded -->
    <div ?hidden=${!familyExpanded}>
      <!-- Repeat per subfamily (skip h3 if subfam.name === null, Pitfall 2) -->
      <div class="pnwm-tb-subfamily-row" style="opacity:0.35 | ''">
        <h3>{Subfamily name}</h3>   ← omitted when name is null
        <div class="pnwm-tb-strip" ...> ... </div>
        <div ?hidden=${!subfamilyExpanded}>
          <!-- Repeat per genus -->
          <div class="pnwm-tb-genus-row" style="opacity:0.35 | ''">
            <h4>
              <button type="button" aria-expanded="false|true">
                {Genus name}
              </button>
            </h4>
            <div class="pnwm-tb-strip" ...> ... </div>
            <!-- Species list — visible only when genus is expanded (BROWSE-05) -->
            <ul ?hidden=${!genusExpanded}>
              <li>
                <a href="/species/{slug}/"><em>{Genus} {species}</em> — {common_name}</a>
              </li>
              ...
            </ul>
          </div>
        </div>
      </div>
    </div>
  </div>
</pnwm-taxon-browser>
```

### Expand/collapse rules (BROWSE-02 through BROWSE-04)

- All families collapsed by default (empty `_expandedFamilies` Set).
- Expanding a family: adds family name to `_expandedFamilies` (new Set, not mutation — Pitfall 6). Family image strip hides when expanded (BROWSE-03 / D-02 implicit: parent images hidden while expanded).
- Expanding a subfamily: adds key to `_expandedSubfamilies`. Subfamily image strip hides when expanded.
- Expanding a genus: adds genus slug to `_expandedGenera`. Genus image strip hides when expanded. Species `<ul>` becomes visible (BROWSE-05).
- Multiple families/subfamilies/genera may be open simultaneously — no mutual exclusion.
- Expand/collapse is instant: `?hidden` attribute toggle. No CSS transitions (D-07).

### Image strip rules (BROWSE-02, BROWSE-06)

- Image strip renders only when `_showImages === true` AND `navImages.length > 0`.
- When a taxon is expanded, its image strip is hidden (parent images hide while children are shown).
- Fixed height: 93px per image. Width: auto (proportional scaling). `object-fit: cover`.
- `alt=""` on all nav thumbnails — decorative at browse level; photographer credit lives on species factsheet (D-04).
- `loading="lazy"` on all nav images to avoid mass image load.
- `flex-shrink:0` prevents images from collapsing when strip overflows on mobile.
- Fallback: `navImages[]` array is pre-computed by `taxon.js` `pickNavImages()` at build time — component renders it as-is without client-side re-sorting (Pitfall 1).

### Show/hide images toggle (BROWSE-07, D-08)

- `<input type="checkbox">` with `<label>` — natively accessible, Pico CSS styles it. Source: RESEARCH.md Alternatives Considered.
- Checked by default (`_showImages = true` in constructor).
- Controls image strip visibility globally across all taxa.
- Rendered inside `render()` in the toolbar div at the top of the component (D-08).

### State filter (SFILT-02, D-05, D-06, D-11)

- `<select>` dropdown rendered inside the toolbar alongside the show/hide toggle.
- Option "All states" (value `""`) is always first and is the default selection.
- State options populated from `_statesAvailable` (derived from `species-states.json`).
- `<select>` is `disabled` until `_statesAvailable.length > 0` — prevents interaction before data loads.
- When a state is selected: taxa with no occurrence records in that state receive `opacity: 0.35` inline style. DOM structure preserved (D-06).
- On fetch error: `_statesAvailable` remains empty, select stays disabled, no error message shown (silent fail — filter simply unavailable).
- `buildStateMap()` and `taxonHasState()` are exported pure functions, unit-testable without DOM.

### Image path convention

Image `src` uses `/images/{filename}` pattern. Must be verified against live species pages before implementation (RESEARCH.md Open Question 1). If the actual path differs (e.g., `/images/{genus_slug}/{filename}`), update this contract and the implementation.

## Interaction States

| State | Element | Visual Treatment |
|-------|---------|-----------------|
| Collapsed family | family row | h2 heading + expand button + image strip visible |
| Expanded family | family row | h2 heading + expand button visible; image strip hidden; children visible |
| Collapsed subfamily | subfamily row | h3 heading (if name not null) + image strip visible |
| Expanded subfamily | subfamily row | h3 heading visible; image strip hidden; genera visible |
| Collapsed genus | genus row | h4 heading + expand button + image strip visible |
| Expanded genus | genus row | h4 heading + expand button visible; image strip hidden; species `<ul>` visible |
| Muted taxon (filtered) | any taxon row | `opacity: 0.35` on row container; expand/collapse still functional |
| Images hidden (toggle off) | all image strips | `display:none` via `?hidden` or conditional render returning `html\`\`` |
| State filter loading | `<select>` | `disabled` attribute present; "All states" shown as only option |
| State filter ready | `<select>` | `disabled` removed; state options populated alphabetically |
| State filter error | `<select>` | Remains `disabled`; stays at "All states" only |

All state transitions are instant — no CSS height, opacity, or clip-path transitions (D-07).

## Accessibility Contract

- `aria-expanded="true|false"` on each expand/collapse `<button>`.
- Heading hierarchy strictly follows noscript block: h2 (family) → h3 (subfamily, conditional) → h4 (genus). No heading levels skipped.
- `alt=""` on all nav images (decorative at browse level).
- `<label for="pnwm-tb-state-filter">` explicitly associated with the state `<select>` via `id`.
- Checkbox toggle has an associated `<label>` wrapping it.
- Muted rows retain `opacity: 0.35` — not `aria-hidden` — so they remain accessible to screen readers (the information "no records in this state" is meaningful).
- Species links use `<em>` for the binomial name inside `<a>` — matches noscript block convention.
