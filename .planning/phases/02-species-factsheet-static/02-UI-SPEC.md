# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Species Factsheet (Static).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — classless CSS framework |
| Preset | not applicable |
| Component library | none — semantic HTML only |
| Icon library | none — text labels only |
| Font | system-ui stack (browser default via Pico CSS) |

**CSS framework:** Pico CSS (`@picocss/pico`) — single stylesheet drop-in.
Source: CONTEXT.md "CSS approach: Classless CSS framework — a single stylesheet drop-in (e.g. Pico CSS or Water.css)."

No custom component classes. All styling achieved via semantic HTML elements (`<nav>`, `<main>`, `<article>`, `<section>`, `<figure>`, `<figcaption>`, `<table>`, etc.) styled by Pico's defaults.

---

## Spacing Scale

Declared values (multiples of 4, expressed as CSS custom properties or inline via Pico defaults):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label spacing |
| sm | 8px | Compact spacing within taxonomy metadata rows |
| md | 16px | Default paragraph and element spacing |
| lg | 24px | Section spacing within species page |
| xl | 32px | Spacing between major page sections (taxonomy, prose, photos, similar species) |
| 2xl | 48px | Top-of-page header clearance, browse page section breaks |
| 3xl | 64px | Not used in Phase 2 |

Exceptions: Image placeholder minimum height 200px (not on 4-point scale, driven by visual legibility).

---

## Typography

Pico CSS sets these via its default stylesheet. The values below are the Pico defaults and must not be overridden in Phase 2.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / metadata | 14px | 400 (regular) | 1.4 |
| Heading (h2 — section) | 20px | 700 (bold) | 1.2 |
| Heading (h1 — species name) | 28px | 700 (bold) | 1.2 |

Notes:
- Species scientific names (`<em>` within `<h1>`) render italic at 28px bold.
- Family and genus headings on browse pages use h2 (20px bold).
- Photographer credit (`<figcaption>`) uses 14px regular.

---

## Color

Pico CSS provides the full palette via CSS custom properties. Phase 2 uses the default Pico light theme without overrides.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `--pico-background-color` | Page background, main content area |
| Secondary (30%) | `#f5f5f5` / `--pico-card-background-color` | Species metadata block, nav bar, browse table rows |
| Accent (10%) | `--pico-primary` (Pico default blue ~`#1b6ef3`) | Navigation links, similar species links, photographer credit links |
| Destructive | not applicable | No destructive actions in Phase 2 |

Accent reserved for: navigation links in `<nav>`, similar species `<a>` elements, photographer credit `<a>` elements. Never used for decorative elements, headings, or body text.

## Component Inventory

These are the semantic HTML structures the executor must implement. No custom CSS classes needed beyond Pico's attribute selectors and element defaults.

### Base Layout (`src/_includes/base.njk`)

```
<html>
  <head>
    <!-- link to pico.min.css -->
  </head>
  <body>
    <nav> … site-wide navigation … </nav>
    <main>
      {% block content %}{% endblock %}
    </main>
  </body>
</html>
```

### Site Navigation

Four links, always present, in order: **Home** → `/`, **Browse** → `/browse/`, **Search** → `/search/`, **Glossary** → `/glossary/`

Search and Glossary link to stub pages with "Coming soon" message until Phase 4.

### Species Factsheet Page (`/species/{slug}/`)

Sections in document order:

1. `<h1><em>{genus} {species}</em></h1>` — scientific name
2. Taxonomy metadata block: common name, NOC ID, authority, family — rendered as `<dl>` (definition list)
3. Prose description — `<article>` wrapping rendered Markdown; omitted entirely if no `.md` file exists
4. Photos — `<figure>` + `<img>` + `<figcaption>` per photo, ordered by `weight`; placeholder block if no images
5. Similar species — `<section>` with `<ul>` of `<a>` links to `/species/{slug}/`; omitted if list is empty

### Image Placeholder

When a species has no images, render:

```html
<figure>
  <div class="no-image-placeholder" aria-label="No images available for this species">
    No photos on file
  </div>
</figure>
```

No broken `<img>` tags. No inline styles.

### Browse Page — All Species (`/browse/`)

Grouped by family, then genus. Structure:

```
<h2>{Family}</h2>
  <h3><a href="/browse/{genus-slug}/">{Genus}</a></h3>
  <!-- repeated per genus -->
```

### Browse Page — Genus Listing (`/browse/{genus-slug}/`)

```
<h1>{Genus}</h1>
<ul>
  <li><a href="/species/{slug}/">{genus} {species} — {common_name}</a></li>
</ul>
```

## Accessibility

- All `<img>` elements have `alt` attribute: `alt="{genus} {species}"` (scientific name of depicted moth)
- Image placeholder `<div>` uses `aria-label="No images available for this species"`
- Nav landmark `<nav>` is present on every page
- `<main>` landmark wraps all page content
- Page `<title>` pattern: `{Genus} {species} — PNW Moths` for species pages; `Browse — PNW Moths` for browse pages
