## Decisions

### Image Data Model (discussed)

- **Storage:** `data/images.csv` — flat CSV, same pattern as `data/species.csv` and `data/records.csv`
- **Columns:** `species_id`, `filename`, `photographer`, `weight` (display order), `license`
- **File location:** `images/{slug}/` — e.g. `images/acronicta-americana/01.jpg`; tracked via Git LFS
- **Build query:** DuckDB joins images to species at build time; template iterates over per-species image array
- **Empty state:** Species with no images display a graceful placeholder (no broken `<img>` tags)

### Base Layout and Styling (discussed)

- **CSS approach:** Classless CSS framework — a single stylesheet drop-in (e.g. Pico CSS or Water.css); no custom classes needed, semantic HTML is enough for a PoC
- **Base template:** Shared Eleventy layout (e.g. `src/_includes/base.njk`) used by all page types
- **Navigation:** Site-wide nav includes all four links — Home, Browse, Search, Glossary — from Phase 2 onward; Search and Glossary point to stub pages that return a "coming soon" message until Phase 4 implements them
- **Nav structure:** `<nav>` in base layout, links use relative paths consistent with slug-based URL structure

### Data Model Gaps — Claude's Discretion

These areas were not discussed; the planner should use these decisions directly:

- **Family field:** Add `family` column to `data/species.csv`. This is the simplest path consistent with the flat-file pattern; no separate lookup table needed. BRWS-01 requires grouping by family then genus — the build query will use this column.
- **Similar species:** Add `similar_species` column to `data/species.csv` as a pipe-separated list of species slugs (e.g. `acronicta-innotata|acronicta-retardata`). Pipe avoids CSV quoting issues. SPEC-04 template iterates this list and renders links to `/species/{slug}/`.
- **Prose Markdown location:** Per-species description files at `content/species/{slug}.md`. Eleventy reads these via a data cascade or a custom data file; the template renders the content if a file exists, silently omits it if not (SPEC-02: "if present").

## Scope Boundary

Phase 2 is static HTML only. No JavaScript, no Vite, no Lit components, no Pagefind. Interactive features (map, chart, slideshow, filters) are Phase 3. Search and glossary pages are Phase 4.

The browse pages (BRWS-01, BRWS-02) are part of Phase 2 — they are static HTML generated by Eleventy, not interactive.
