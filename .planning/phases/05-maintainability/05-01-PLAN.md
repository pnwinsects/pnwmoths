**1. `.github/workflows/deploy.yml`** — Build + deploy on push to main (per D-01, D-02):

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: nschloe/action-cached-lfs-checkout@v1
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
      - run: npm ci
      - name: Cache lychee binary
        uses: actions/cache@v4
        with:
          path: ~/.local/bin/lychee
          key: lychee-0.23.0-linux-amd64
      - name: Install lychee if not cached
        run: |
          if [ ! -f ~/.local/bin/lychee ]; then
            mkdir -p ~/.local/bin
            wget -qO /tmp/lychee.tar.gz \
              https://github.com/lycheeverse/lychee/releases/download/lychee-v0.23.0/lychee-x86_64-unknown-linux-gnu.tar.gz
            tar -xzf /tmp/lychee.tar.gz -C ~/.local/bin
            chmod +x ~/.local/bin/lychee
          fi
          echo "$HOME/.local/bin" >> $GITHUB_PATH
      - name: Cache lychee URL results
        uses: actions/cache@v4
        with:
          path: .lycheecache
          key: lychee-cache-${{ github.run_id }}
          restore-keys: lychee-cache-
      - uses: actions/configure-pages@v5
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: _site

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment
```

Per D-03: LFS is handled by `nschloe/action-cached-lfs-checkout` which runs `git lfs install` + `git lfs pull` with caching.
Per D-11: Caches npm (via setup-node), LFS objects (via cached-lfs-checkout), lychee binary, and lychee URL results.

Create two files (per D-08, D-09, D-10):

**1. `Dockerfile`** — Multi-stage build. Stage 1 downloads lychee binary. Stage 2 is the runtime:

```dockerfile
# Stage 1: download lychee binary
FROM debian:bookworm-slim AS lychee-builder
RUN apt-get update && apt-get install -y --no-install-recommends ca-certificates wget \
    && rm -rf /var/lib/apt/lists/*
ARG LYCHEE_VERSION=0.23.0
RUN wget -qO /tmp/lychee.tar.gz \
    "https://github.com/lycheeverse/lychee/releases/download/lychee-v${LYCHEE_VERSION}/lychee-x86_64-unknown-linux-gnu.tar.gz" \
    && tar -xzf /tmp/lychee.tar.gz -C /usr/local/bin \
    && chmod +x /usr/local/bin/lychee

# Stage 2: runtime image
FROM node:22-bookworm-slim
RUN apt-get update && apt-get install -y --no-install-recommends \
    git git-lfs ca-certificates \
    && git lfs install \
    && rm -rf /var/lib/apt/lists/*
COPY --from=lychee-builder /usr/local/bin/lychee /usr/local/bin/lychee
WORKDIR /workspace
```

Key points:
- `node:22-bookworm-slim` matches `.nvmrc` (Node 22) per D-10.
- `git-lfs` installed via apt on bookworm (verified available).
- `@duckdb/node-api` pre-built binaries for linux/amd64 are pulled by `npm ci` inside the container.
- `pagefind` is in devDependencies and self-resolves its binary via npm.
- No COPY of project files — designed for volume-mount dev use per D-08.

**2. `docker-compose.yml`** — Convenience wrapper for the dev environment:

```yaml
services:
  dev:
    build: .
    volumes:
      - .:/workspace
      - node_modules:/workspace/node_modules
    working_dir: /workspace
    command: bash
    stdin_open: true
    tty: true

volumes:
  node_modules:
```

Usage pattern for maintainers:
- `docker compose run --rm dev npm ci` — install dependencies
- `docker compose run --rm dev npm run build` — full build
- `docker compose run --rm dev bash` — interactive shell

The named `node_modules` volume avoids host/container architecture conflicts with native binaries (DuckDB).
