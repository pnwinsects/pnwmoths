| Boundary | Description |
|----------|-------------|

| Light DOM `<figure>` → component | Image metadata (alt, locality, photographer, dataset-*) is read by `connectedCallback` from same-origin server-rendered HTML — no cross-origin or user-input source |

## STRIDE Threat Register

This phase is UI-only: no network requests originate from the modified code, no user-submitted data enters the component, no auth/session/access-control surface exists, no crypto. ASVS L1 categories V2–V6 do not apply.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering (XSS) | `html\`<img src=${img.src} alt=${img.alt}>\`` inside `.thumbnail` button | accept | Lit's tagged-template engine escapes interpolations into attribute context by default. Sources (`src`, `alt`) come from static build-time HTML emitted by Eleventy from project-controlled data — not from user input. No additional sanitization needed. |
| T-23-02 | Information Disclosure | Decorative `<img alt="">` inside `.thumbnail` | accept | Empty alt is the correct accessibility pattern for decorative imagery whose label is carried by the wrapping `<button aria-label="…">` (Pitfall 5). No PII concern. |
| T-23-03 | Denial of Service (browser) | ResizeObserver callback in `firstUpdated` | mitigate | The callback uses the guard pattern `if (overflows !== this._stripOverflows)` to prevent infinite re-render loops (Pitfall 3). Without the guard, a layout-affecting re-render could fire the observer indefinitely. Verified by Task 3 acceptance criterion (presence of guard in source). |
| T-23-SC | Tampering (supply chain) | npm/pip/cargo installs | accept | No new packages introduced in this phase (RESEARCH "Standard Stack" section). Package Legitimacy Audit not applicable. |
