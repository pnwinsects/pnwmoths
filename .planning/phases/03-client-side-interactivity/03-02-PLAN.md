**Step 2: Create pnwm-image-slideshow.js**

- Import LitElement, html, css from 'lit'.
- Class `PnwmImageSlideshow extends LitElement`.
- Static properties: `slug` (String), `_currentIndex` (Number, state: true), `_lightboxOpen` (Boolean, state: true), `_images` (attribute: false, state: true).
- Constructor: slug='', _currentIndex=0, _lightboxOpen=false, _images=[].
- `connectedCallback()`: call super. Query slotted/child `<figure>` elements from light DOM children. Extract image data from each figure: src from img, alt from img, photographer from figcaption text. Store in _images array. If no figures found, _images stays empty.
- Static styles:
  - `:host { display: block; }`.
  - `.slideshow { position: relative; }`.
  - `.slide { text-align: center; }`.
  - `.slide img { max-width: 100%; height: auto; cursor: pointer; }`.
  - `.controls { display: flex; justify-content: center; align-items: center; gap: 8px; margin-top: 8px; }`.
  - `.controls button { min-width: 44px; min-height: 44px; }` (touch target per UI-SPEC).
  - `.dots { display: flex; gap: 4px; }`.
  - `.dot { width: 8px; height: 8px; border-radius: 50%; background: var(--pico-muted-color); }`.
  - `.dot.active { background: var(--pico-primary); }`.
  - Lightbox: `.lightbox { position: fixed; inset: 0; background: rgba(0,0,0,0.85); display: flex; align-items: center; justify-content: center; z-index: 1000; }`.
  - `.lightbox img { max-width: 90vw; max-height: 90vh; object-fit: contain; }`.
  - `.lightbox-close { position: absolute; top: 16px; right: 16px; width: 44px; height: 44px; background: transparent; border: none; color: white; font-size: 24px; cursor: pointer; }`.
- `render()`:
  - If _images.length === 0: render the light DOM children as-is (the fallback figures from the template). Use `html\`<slot></slot>\``.
  - If _images.length === 1: Single image, no controls. Image is clickable to open lightbox.
  - If _images.length > 1: Show current image, prev/next buttons with aria-labels ("Previous photo", "Next photo" per UI-SPEC), dot indicators, index label "N of M".
  - Clicking image calls `_openLightbox()`.
  - Per UI-SPEC: `role="region"` with `aria-label="Species photos"` on container.
- `_openLightbox()`: Set _lightboxOpen=true. Set `inert` attribute on `document.querySelector('main')` for focus trap. Focus close button.
- `_closeLightbox()`: Set _lightboxOpen=false. Remove `inert` from main.
- Lightbox render: dark overlay, centered image at up to 90vw/90vh, close button top-right (44x44px, "X"), aria-label="Close lightbox". Escape key closes (add keydown listener in connectedCallback, remove in disconnectedCallback).
- `_prev()`, `_next()`: Cycle _currentIndex with wrap-around.
- When rendered with images, hide the light DOM children (the static figures) using a style trick: in connectedCallback, set `this.querySelectorAll(':scope > figure').forEach(f => f.style.display = 'none')` so the static fallback is hidden when JS runs.

**Step 4: Add filter wiring to species.njk**

At the end of `src/species/species.njk`, add an inline script that connects the filter-bar events to map and chart:

```html
<script type="module">
  // Wire filter-bar events to map and chart components
  const map = document.querySelector('pnwm-occurrence-map');
  const chart = document.querySelector('pnwm-phenology-chart');

  document.addEventListener('pnwm-filter-change', (e) => {
    if (map) map.filters = e.detail;
    if (chart) chart.filters = e.detail;
  });
</script>
```

This inline script listens for the composed pnwm-filter-change event (which bubbles up from pnwm-filter-bar through shadow DOM) and sets the filters property on both map and chart, triggering their reactive update cycle.
