# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Pico CSS v2.1.1 — existing) |
| Preset | not applicable |
| Component library | Lit custom elements (web components) |
| Icon library | none — use Unicode or inline SVG for close/expand only |
| Font | System-ui stack (Pico CSS default: system-ui, Segoe UI, Roboto, Helvetica, Arial) |

**Note:** This project uses Pico CSS (`@picocss/pico ^2.1.1`) as its base styling framework, established in Phase 2. shadcn is not applicable — the project is Eleventy + Lit, not React/Next.js. All new interactive components are Lit custom elements that inherit Pico CSS custom properties from the host page.

**Vite integration:** Phase 3 adds Vite as a JS bundler for Lit components only. Pico CSS continues to be served as a passthrough copy, not processed by Vite. Lit components use CSS-in-JS (tagged template literals) for component-scoped styles that extend, not override, Pico tokens.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip spacing inside filter bar |
| sm | 8px | Filter control internal padding, chart axis labels |
| md | 16px | Default spacing between components on factsheet, map/chart padding |
| lg | 24px | Section gaps between map, chart, and filter regions |
| xl | 32px | Gap between factsheet sections (static content and interactive zone) |
| 2xl | 48px | Not used in Phase 3 interactive components |
| 3xl | 64px | Not used in Phase 3 interactive components |

Exceptions:
- Leaflet map container minimum height: 320px (overrides spacing scale — fixed height for usability)
- Touch targets for filter controls: minimum 44px height (accessibility requirement)
- Lightbox close button: 44x44px touch target

## Typography

All sizes are relative to Pico CSS base (16px at mobile, scaling to 20px at 1280px+ via Pico responsive font-size). Sizes below are at 16px base.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 (regular) | 1.5 |
| Label | 14px (0.875rem) | 400 (regular) | 1.4 |
| Heading (section, e.g. "Occurrence Map") | 20px (1.25rem) | 700 (bold) | 1.2 |
| Display (species scientific name h1) | 32px (2rem) | 700 (bold) | 1.125 |

**Interactive component typography:**
- Filter control labels: 14px / 400 / 1.4 (same as Label)
- Chart axis tick labels: 12px / 400 / 1.0 (exception for data density)
- Chart bar labels (count): 12px / 600 (semibold) / 1.0
- Map popup species name: 14px / 600 (semibold) / 1.3
- Map popup metadata: 12px / 400 / 1.4
- Loading state text: 14px / 400 / 1.5, muted color

## Color

The project inherits Pico CSS CSS custom properties. Interactive components reference these tokens; do not hardcode hex values in Lit component styles.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--pico-background-color)` (#fff light / #13162 dark) | Page background, map container background |
| Secondary (30%) | `var(--pico-card-background-color)` + `var(--pico-muted-border-color)` | Filter bar container, chart container, occurrence table |
| Accent (10%) | `var(--pico-primary)` (#0172ad) | Active filter chips, selected bar in phenology chart, map occurrence point fill, lightbox open-state button |
| Destructive | `var(--pico-del-color)` (Pico red) | Not applicable in Phase 3 (no destructive actions) |

Accent reserved for: active/selected state of filter chips, the currently-highlighted phenology chart bar on hover, Leaflet map point markers, and the "View larger" button hover state. Accent is NOT used for static text, nav links, or container borders.

**Leaflet map markers:** Use `var(--pico-primary)` (#0172ad) as fill with 0.7 opacity for individual occurrence points. Clustered markers use the same color at full opacity.

**Loading state:** Use `var(--pico-muted-color)` (#646b79) for spinner and loading text.

**JS-off fallback table:** Standard Pico table styling — no custom colors required.

## Component Inventory

Each interactive component is a Lit custom element. The HTML fallback (for JS-off) is declared in the same element slot and hidden when JS runs.

| Component | Custom Element Tag | Fallback (JS-off) |
|-----------|-------------------|-------------------|
| Occurrence map | `<pnwm-occurrence-map>` | Hidden (static occurrence table `<pnwm-occurrence-table>` shown instead) |
| Phenology chart | `<pnwm-phenology-chart>` | Static `<table>` with month/count rows |
| Filter bar | `<pnwm-filter-bar>` | Hidden entirely (table is unfiltered) |
| Image slideshow | `<pnwm-image-slideshow>` | Static `<figure>` list (all images visible) |
| Occurrence table (fallback) | `<pnwm-occurrence-table>` | Always rendered as `<table>` in HTML; JS component adds pagination only |

**Progressive enhancement pattern:** All `<pnwm-*>` elements render their `<slot>` fallback content by default. The Lit component's `connectedCallback` hides the fallback and renders the interactive version. A `<noscript>` block shows a note: "Enable JavaScript to view the interactive map and filters."

## Interaction States

### Occurrence Map (`<pnwm-occurrence-map>`)

| State | Visual |
|-------|--------|
| Loading | Spinner centered in map container (320px min-height); text "Loading occurrence data…" at muted color |
| Loaded | Leaflet map with point markers; zoom controls top-left |
| Empty (no records match filter) | Map shows region with no markers; banner inside map: "No records match the current filters." |
| Error (Parquet fetch failed) | Red banner inside map container: "Could not load occurrence data. Try reloading the page." |
| Marker hover | Tooltip shows: locality, state, year, record type |
| Marker click | Popup shows: locality, state, county, year, month/day (if available), collector, record type |

### Phenology Chart (`<pnwm-phenology-chart>`)

| State | Visual |
|-------|--------|
| Loading | Skeleton bars (muted color, no animation) in chart area |
| Loaded | 12 vertical bars, one per month (Jan–Dec), bar height proportional to record count |
| Empty | All bars at zero height; x-axis labels remain; text below: "No records match the current filters." |
| Bar hover | Bar fills to accent color; tooltip shows month name + count |

### Filter Bar (`<pnwm-filter-bar>`)

| State | Visual |
|-------|--------|
| No filters active | All selects at default value; no active chips shown |
| Filter active | Chip appears below selects showing active value; "Clear" link (16px, muted color) at end of chips |
| Updating | Map and chart show loading spinner; filter bar remains interactive |

Filter controls:
- State: `<select>` — "All states" default, alphabetical PNW state list
- Record type: `<select>` — "All types" default
- Year range: dual `<input type="range">` sliders — min 1900, max current year, step 1; labels show selected range as "YYYY – YYYY"

### Image Slideshow (`<pnwm-image-slideshow>`)

| State | Visual |
|-------|--------|
| Single image | No prev/next controls shown; just image with caption |
| Multiple images | Previous/next arrow buttons (44px touch target); dot indicators; current index label "2 of 7" |
| Lightbox closed | Normal slideshow |
| Lightbox open | Dark overlay (rgba 0,0,0,0.85); image centered at up to 90vw / 90vh; close button top-right (44x44px, X icon) |
| No images | Pico-styled `<figure>` with placeholder: "No photos on file for this species." |

## Loading Strategy

- Parquet file fetch begins on component `connectedCallback` (not on user interaction)
- hyparquet loads the per-species Parquet file from `/species/{slug}/{slug}.parquet`
- A single fetch/parse is shared between map and chart via a module-level cache keyed by slug
- Loading state is shown per-component independently (map and chart each show their own spinner)
- If fetch takes > 8 seconds, show error state with reload prompt

## Accessibility Contract

- All custom elements must publish `role` attributes where native semantics are absent
- Map container: `role="application"` with `aria-label="Occurrence map for {species name}"`
- Chart container: `role="img"` with `aria-label="Phenology chart for {species name}"` (fallback table serves as accessible alternative)
- Filter selects: use `<label>` elements with explicit `for`/`id` pairing — not `aria-label`
- Slideshow: `role="region"` with `aria-label="Species photos"`; arrow buttons have explicit `aria-label`
- Lightbox: focus trap when open; `Escape` key closes; first focus goes to close button
- Tab order: filter bar → map → chart → occurrence table → slideshow (matches visual top-to-bottom order)
