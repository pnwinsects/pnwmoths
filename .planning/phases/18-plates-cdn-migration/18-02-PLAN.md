<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer terminal → bunny.net Storage API | BUNNY_API_KEY sent as HTTP header during upload |
| CDN → browser | Tiles served as public static JPEG files; no auth required for reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05 | Information Disclosure | BUNNY_API_KEY in terminal | mitigate | Key passed as env var (`BUNNY_API_KEY="$BUNNY_API_KEY" node ...`); never echoed or logged by the script; shell history may retain it — user should clear after use (`history -d`) |
| T-18-06 | Tampering | CDN upload — accidental delete | accept (not applicable) | Script uses HTTP PUT only; no DELETE calls; no sync operation; re-running is safe (PUT is idempotent) |
| T-18-07 | Availability | Partial upload failure | accept | HTTP PUT is idempotent; failed upload can be re-run from scratch; bunny.net PUT overwrites safely; risk is upload time only (15-30 min) |
</threat_model>
