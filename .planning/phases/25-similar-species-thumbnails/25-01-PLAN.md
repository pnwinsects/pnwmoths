## Trust Boundaries

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering / Injection (XSS) | `species.njk` similar-species section rendering `s.common_name`, `s.genus`, `s.species`, `slug`, and `img.filename` into HTML attributes and text | mitigate | Nunjucks auto-escaping is enabled by default (Eleventy default config; not disabled in `eleventy.config.js`) — all `{{ ... }}` interpolations are HTML-escaped, so `&`, `<`, `>`, `"` in data values cannot break out of attribute or text contexts. No `\| safe` filter is used in this section. Curator-controlled CSV data is the only source. |
| T-25-02 | Tampering (URL injection) | CDN `<img src="{{ cdnBaseUrl }}/{{ slug }}/{{ simImages[0].filename \| urlencode }}">` | mitigate | `cdnBaseUrl` is a hard-coded constant in `eleventy.config.js` line 16, not user-influenced. `slug` is a controlled identifier (lowercase alpha + hyphens by CSV convention). Filename is passed through `\| urlencode` (encodeURIComponent), so a maliciously crafted filename containing `../` or query strings cannot escape the CDN path segment. |
| T-25-03 | Tampering (URL injection on internal link) | `<a href="{{ ('/species/' + slug + '/') \| url }}">` | accept | `slug` is a build-time-controlled identifier from `species.csv` (curator-trusted). Nunjucks auto-escapes the attribute, and `\| url` only prefixes pathPrefix. A malicious slug would be visible during the build (filenames must match the page-generation slug) and is out of scope for an attacker. |
| T-25-04 | Denial of service (image fetch storms) | Below-fold thumbnails fetched from bunny.net CDN on every species page view | accept | `loading="lazy"` defers image fetches until scroll, reducing initial-page-load load. CDN caching handles the steady-state traffic; bunny.net Pull Zone is sized for production traffic per Phase 13. |
| T-25-05 | Information disclosure | None — section contains only public taxonomic data | accept | All data displayed (species names, slugs, thumbnails) is already public on existing species pages. No PII, no user accounts, no auth context on this site. |
| T-25-SC | Tampering (supply chain) | npm/pip/cargo installs | mitigate | No package installs in this phase (template + CSS only). Threat is N/A for this scope but kept in register for ASVS L1 completeness. |

## Residual Risk

Low. Phase modifies one template block and appends one CSS rule set. No new external dependencies, no JS execution, no user input handling, no auth surface. Nunjucks auto-escaping and the existing `| urlencode` / `| url` filter discipline cover all data-flow points.
