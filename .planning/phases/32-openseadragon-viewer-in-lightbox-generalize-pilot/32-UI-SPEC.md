Accent (`#a4ab78`) is reserved for:

OSD viewer background: `#111` — dark canvas, intentional for specimen photo viewing.

Components modified in this phase:

**New elements added:**

| Element | CSS class | States | Notes |
|---------|-----------|--------|-------|
| In-lightbox prev button | `.lightbox-prev` | default, hover, focus | Only rendered when `useOsd && _highResSpecimens.length > 1` |

| Specimen caption | `.caption-line` (existing) | updates reactively on `_currentIndex` change | "Specimen {id} · Dorsal/Ventral" |

**Prev/next button specification:**
- Size: 44px × 44px minimum (touch target rule)
- Position: vertically centered inside lightbox, left and right of OSD viewer
- Background: `rgba(0, 0, 0, 0.5)` (semi-transparent dark — consistent with lightbox overlay tone)
- Color: `#ffffff`
- Font size: 20px (Unicode arrows &#x276E;/&#x276F; or &#x2039;/&#x203a;)
- Border: none
- z-index: above OSD viewer (OSD viewer has no explicit z-index; buttons inherit lightbox stacking context at z-index 9000)
- Aria-label: "Previous specimen" / "Next specimen"

**Close button investigation (folded from Phase 23):**
- Existing selector: `.lightbox-close` with `position: absolute; top: 16px; right: 16px`
- Current behavior: button exists in shadow DOM, `@click=${() => this._closeLightbox()}` is wired
- Investigation target: Verify z-index layering within Lit shadow DOM + OSD element stacking; confirm `focusable` close button receives pointer events above the OSD canvas
- No visual change to close button spec — 44px × 44px, white ×, top-right of lightbox — if it is broken, the fix is pointer-event or z-index only

### `species.njk` (Nunjucks template)

**New figure elements for high-res species:**

```
{% elif highResEntry and highResEntry.high_res_available %}
  {% for specimen in highResEntry.specimens %}
    <figure>
      <img src="{cdnBaseUrl}/{specimen.tiles_path}/{specimen.specimen_id}-{specimen.view}_files/0/0_0.webp"
           alt="{sp.genus} {sp.species}">
      <figcaption>Specimen {specimen.specimen_id} · {Dorsal|Ventral}</figcaption>
    </figure>
  {% endfor %}
```

- Image src: level-0 DZI tile (`_files/0/0_0.webp`) — single-tile coarsest resolution, already on CDN
- figcaption: "Specimen {specimen_id} · Dorsal" or "Specimen {specimen_id} · Ventral" (D → Dorsal, V → Ventral)
- Figure order matches `highResEntry.specimens` array order so `_images[i]` aligns with `_highResSpecimens[i]`
- No-JS fallback: figures are visible static `<img>` elements until the Lit component's `connectedCallback` hides them

## Interaction Contract

### High-res species path (VIEWER-01, VIEWER-03, VIEWER-04)

1. Page loads → component reads `<figure>` elements from light DOM into `_images[]`
2. Component renders: main slide (level-0 tile img) + thumbnail strip (same tile URL per thumbnail)
3. User clicks main slide → `_openLightbox()` → OSD initializes with current specimen's DZI URL
4. If `_highResSpecimens.length > 1`: prev/next buttons visible inside lightbox
5. Prev/next click → increment/decrement `_currentIndex` (wrapping) → `viewer.open(newDziUrl)` — no destroy/recreate
6. Caption line updates reactively: "Specimen {id} · Dorsal/Ventral"
7. Close: click overlay background, click ×, or press Escape → `_closeLightbox()` → `viewer.destroy()`

### Low-res species path (VIEWER-02)

1. Page loads → existing Phase 23 static `<img>` figures in light DOM
2. Component renders: main slide + thumbnail strip (unchanged)
3. User clicks main slide → `_openLightbox()` → static `<img>` in lightbox (no OSD)
4. No prev/next navigation buttons (no `useOsd` flag)
5. Close: same as above

### No-photos path (existing)

- Template renders: `<figure><div aria-label="No images available for this species">No photos on file for this species.</div></figure>`
- Component sees no `<img>` in figures → `_images.length === 0` → falls through to `<slot>` render

## Accessibility Contract

- All interactive buttons: `min-width: 44px; min-height: 44px` (touch target)
- Lightbox focus trap: sibling-walk `inert` on ancestor siblings (existing Phase 28 pattern — do not break)
- Close button receives focus on `_openLightbox()` (existing)
- Escape key closes lightbox (existing `_handleKeydown`)
- OSD viewer: `aria-label` not required (OSD handles its own accessibility internally); lightbox container has no explicit role beyond being in focus trap
- Carousel: `role="tablist"` + `role="tab"` + `aria-selected` on thumbnails (existing — do not change)
- `data-pagefind-ignore` on `<pnwm-image-slideshow>` suppresses lightbox content from Pagefind index (existing — do not change)
