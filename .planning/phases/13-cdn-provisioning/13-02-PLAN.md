const DEFAULT_MOTHS_SOURCE = '/Users/rainhead/dev/pnwinsects-app/pnwmoths_https/usr/local/www/pnwmoths/django/pnwmoths/static/media/moths';
const DEFAULT_GLOSSARY_SOURCE = '/Users/rainhead/dev/pnwinsects-app/pnwmoths_https/usr/local/www/pnwmoths/django/pnwmoths/static/media/glossary-images';

**parseMotFilename(fname):** Extract Genus and species from Django filename.
- Pattern: `/^([A-Z][a-z]+) ([a-z]+)-/`
- Returns `{ genus, species }` or `null` for non-matching filenames (numeric IDs like `0817021.JPG`)
- Non-matching filenames: log `[migrate-images] Skipping unparseable filename: ${fname}` and return null

**toSlug(genus, species):** `return \`${genus}-${species}\`.toLowerCase();`

**parseViewSpecimen(fname):** Extract view/specimen from filename suffix.
- Pattern: match `-([A-Z])-([A-Z])\.` at end of basename (e.g. `-A-D.jpg` → specimen=A, view=dorsal)
- View mapping: D → dorsal, V → ventral, L → lateral, H → head; unknown letters → raw letter
- Returns `{ view, specimen }` or `{ view: '', specimen: '' }` if no match

**main() function:**

1. **Load species lookup** — read `data/species.csv`, build `slug → species_id` map. Fatal if file missing.

2. **Load Django DB export** — read `DEFAULT_DB_EXPORT` CSV (columns: `original_filename`, `photographer`, `license`). This is a CSV exported by the project owner before running this script. If the file is missing, log a warning and proceed with empty photographer/license (empty strings) — do NOT fatal exit, so the script can produce a skeleton CSV for review.

3. **Scan moths/ directory** (species photos):
   - `readdir(MOTHS_SOURCE, { withFileTypes: true })`
   - Filter to `e.isFile()` only (skips thumbnail/, medium/, cache/ subdirs)
   - For each file: call `parseMotFilename(e.name)`. If null → warn and skip.
   - Group files by slug. For each slug, sort files alphabetically and assign `weight` starting at 1.
   - Build output rows with schema: `{ species_slug, filename, photographer, weight, license, view, specimen, navigational: '' }`
   - `navigational` is always blank (not 'true') — nav images are curated manually after import

4. **Scan glossary-images/ directory** (glossary images):
   - `readdir(GLOSSARY_SOURCE, { withFileTypes: true })`
   - Filter to `e.isFile()` only
   - Skip files whose extension (lowercased) is not `.jpg`, `.jpeg`, `.png`, `.gif`
   - Each glossary image: `species_slug = 'glossary'`, filename = original filename, weight = 1, navigational = ''
   - Photographer/license from DB export lookup by original_filename; empty string if not found

5. **Upload via rclone** (if `DRY_RUN !== true`):
   - For each unique slug dir in species photos:
     ```js
     execFileSync('rclone', [
       'copy', '--ignore-times',
       MOTHS_SOURCE,
       `${RCLONE_REMOTE}/${slug}/`
     ], { stdio: 'inherit' });
     ```
     Note: upload each file individually using:
     ```js
     execFileSync('rclone', [
       'copy', '--ignore-times',
       join(MOTHS_SOURCE, fname),
       `${RCLONE_REMOTE}/${slug}/`
     ], { stdio: 'inherit' });
     ```
   - For glossary images:
     ```js
     execFileSync('rclone', [
       'copy', '--ignore-times',
       GLOSSARY_SOURCE,
       `${RCLONE_REMOTE}/glossary/`
     ], { stdio: 'inherit' });
     ```
   - If `DRY_RUN === true`: log each rclone command that would run instead of executing

6. **Write images.csv**:
   ```js
   const COLUMNS = ['species_slug','filename','photographer','weight','license','view','specimen','navigational'];
   const csvOut = stringify(outputRows, { header: true, columns: COLUMNS });
   await writeFile('data/images.csv', csvOut);
   console.log(`[migrate-images] Wrote ${outputRows.length} rows to data/images.csv`);
   ```

7. **Graceful exit if sources missing**:
   - If MOTHS_SOURCE does not exist: `console.warn('[migrate-images] Moths source not found — skipping'); process.exit(0);`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Django local filesystem → migrate-images.js | Files read from developer's local pnwinsects-app directory |
| migrate-images.js → rclone → bunny.net FTP | Credentials in rclone config file; files uploaded over FTP |
| migrate-images.js → data/images.csv | Script writes CSV that the build pipeline ingests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02-01 | Tampering | rclone copy deletes CDN bucket files | mitigate | Script uses `rclone copy` exclusively (never `rclone sync`). Grep check in acceptance criteria confirms no `sync` call. PITFALLS.md Pitfall 3 documents the risk. |
| T-13-02-02 | Information Disclosure | bunny.net FTP password in rclone config | accept | rclone stores obscured password in `~/.config/rclone/rclone.conf` (not in repo). Developer machine only. No secret in codebase. |
| T-13-02-03 | Tampering | mtime diffing silently skips re-uploads | mitigate | `--ignore-times` flag forces content-based comparison. Documented in UPLOADING_IMAGES.md (Plan 04). |
| T-13-02-04 | Denial of Service | Script overwrites images.csv with partial data if interrupted mid-run | mitigate | Write to images.csv is a single atomic `writeFile` call at the end of main() after all processing completes. Partial runs leave the old file intact. |
| T-13-02-05 | Spoofing | Malicious filename in Django moths/ dir (path traversal) | mitigate | `parseMotFilename` regex `^([A-Z][a-z]+) ([a-z]+)-` rejects filenames not starting with Genus Species pattern. Glossary filenames have extension-only filter. Neither uses the filename in shell exec directly — rclone receives the full source path. |
