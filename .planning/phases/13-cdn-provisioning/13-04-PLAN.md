# Task: Upload Images to the CDN

## What This Changes
- `bunny.net` Storage Zone `pnwmoths` — new image files added under `{slug}/{original-filename}` (species) or `glossary/{original-filename}` (glossary)
- `data/images.csv` — new rows referencing the uploaded images
- CDN cache — may need explicit purge when replacing an existing file

## Before You Start

You will need:
- **FTP password** for the `pnwmoths` Storage Zone — request from the project owner
- **bunny.net API key** — request from the project owner (needed only for cache invalidation)
- `rclone` installed: `brew install rclone` (macOS) or see https://rclone.org/install/

## Steps

### 1. Configure rclone (first time only)

```bash
rclone config
```

At the prompts:
- Name: `bunny`
- Type: `ftp`
- Host: `ny.storage.bunnycdn.com`
- User: `pnwmoths`
- Password: run `rclone obscure YOUR_FTP_PASSWORD` and paste the output
- Port: `21` (press Enter for default)
- All other settings: press Enter to accept defaults

Verify the connection:
```bash
rclone ls bunny:pnwmoths/
# Expected: lists species slug directories. An error means wrong credentials.
```

### 2. Prepare the image file

- Species photos go in a directory named after the species slug (e.g. `acronicta-americana/`)
- Keep the original filename as-is — do not rename images
- Filenames may contain spaces (e.g. `Acronicta americana-A-D.jpg`) — this is correct

### 3. Upload new images

```bash
# Upload a new species photo (replace slug and filename):
rclone copy --ignore-times \
  "/path/to/Acronicta americana-A-D.jpg" \
  "bunny:pnwmoths/acronicta-americana/"

# Upload a glossary image:
rclone copy --ignore-times \
  "/path/to/my-glossary-image.jpg" \
  "bunny:pnwmoths/glossary/"
```

Use `--ignore-times` always. bunny.net's FTP interface does not support modification times — without this flag, rclone may skip uploads silently.

### 4. Update data/images.csv

Add a row to `data/images.csv` for the new image:

| Field | Value |
|-------|-------|
| species_slug | e.g. `acronicta-americana` |
| filename | original filename including spaces, e.g. `Acronicta americana-A-D.jpg` |
| photographer | Photographer's name |
| weight | Display order (lower = earlier); use next integer after existing rows for this slug |
| license | CC license or `© Photographer Name` |
| view | `dorsal`, `ventral`, `lateral`, or `head` (optional) |
| specimen | Specimen letter: `A`, `B`, `C`, `D` (optional) |
| navigational | Leave blank unless this is a curated navigation image |

### 5. Invalidate CDN cache (when replacing an existing file only)

When replacing an image that was already served by the CDN, the cached version may be stale for hours or days. Purge it:

```bash
# Replace FILENAME with the URL-encoded path (spaces → %20):
curl -X POST "https://api.bunny.net/purge?url=https://pnwmoths.b-cdn.net/acronicta-americana/Acronicta%20americana-A-D.jpg" \
  -H "AccessKey: YOUR_BUNNY_API_KEY"
```

For new uploads (first time a filename appears), no cache invalidation is needed.

## Verify

- Run `rclone ls bunny:pnwmoths/{slug}/` — the new file should appear in the listing
- Open `https://pnwmoths.b-cdn.net/{slug}/{url-encoded-filename}` in an incognito browser — expect HTTP 200 and the image displayed
- Run `npm run build` — should complete without errors (build-data.js validates the new images.csv row)

## WARNING: Never Use rclone sync

`rclone sync` makes the destination match the source exactly — **it deletes files in the bucket that are not present in your local source directory**. A partial local directory + rclone sync = empty production bucket.

Always use `rclone copy`. Only use `rclone sync --dry-run` (with the `--dry-run` flag) if you need to audit what a sync would do, and review the output carefully before removing the flag.
