<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dropbox API → dropbox-list.js | Untrusted HTTP responses (Dropbox-controlled) parsed as JSON; treated as data, never executed. |
| Manifest rows → CSV file | Untrusted filename strings written to a committed CSV; binomial_raw and filename_raw fields could contain CSV-special chars. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26.02-01 | Information disclosure | dropbox-list.js error paths | mitigate | Library never references DROPBOX_TOKEN in error messages; the only place the token appears is the Authorization header on outgoing requests. Dropbox does not echo the Authorization header back in error response bodies. |
| T-26.02-02 | Tampering | CSV injection via Dropbox-supplied filename_raw / binomial_raw | mitigate | csv-stringify (already in deps, used here) automatically quotes any field containing comma, newline, double-quote, or other CSV-special chars. The `columns: COLUMNS` option also pins schema. Verified by the round-trip test asserting a comma-containing field survives intact. |
| T-26.02-03 | Denial of service | infinite pagination loop on Dropbox quirk | mitigate | Loop terminates strictly when `has_more === false`. No retry-here logic. If Dropbox returned `has_more: true` with empty entries indefinitely, the loop would still bound by the rate-limited HTTP-call budget; Plan 03's `withRetry` (D-15) caps at 5 attempts so a wedged page surfaces as a manifest failure row. |
| T-26.02-04 | Repudiation | INVESTIGATION_BUCKETS list drifting from D-12 | mitigate | INVESTIGATION_BUCKETS is a module-level const inside manifest.js, asserted by a unit test against the literal `['genus-only', 'likely-synonym', 'provisional', 'unparseable']` set. A future refactor that drops `provisional` from this list would fail the test. |
| T-26.02-SC | Tampering | No new package-manager installs in this plan | accept | This plan adds no new dependencies; csv-parse@^6.2.1 + csv-stringify@^6.7.0 are already in package.json (Phase 13 introduced them). No new supply-chain risk. |
</threat_model>
