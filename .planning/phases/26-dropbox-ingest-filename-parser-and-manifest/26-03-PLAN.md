## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars → script | DROPBOX_TOKEN provided at invocation; never persisted to disk by this script. |
| Dropbox API → script | Untrusted response bodies parsed as JSON. |
| Script → committed manifest CSV | Dropbox-supplied filenames cross into a committed file. |
| Script → console output | Error messages may contain the token unless redacted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26.03-01 | Information disclosure | DROPBOX_TOKEN in error messages or logs | mitigate | Every error path is wrapped by withRetry which applies the redact-token idiom from scripts/upload-plates.js:112. The withRetry helper handles the empty-token edge case to avoid the `new RegExp('', 'g')` corruption pitfall.
